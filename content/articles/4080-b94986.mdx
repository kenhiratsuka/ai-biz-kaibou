# 学習塾・予備校向け生徒評価シート・進路指導報告書作成代行

## なぜ今なのか

日本の学習塾・予備校市場は、2023年時点で約1兆円規模（矢野経済研究所調べ）。全国に約5万校以上の学習塾が存在し、その大半が中小・個人経営です。大手は除き、地域密着型の塾講師の多くは「授業以外の事務作業」に慢性的に疲弊しています。

その最大の「隠れた業務負担」が、**生徒評価シートと進路指導報告書の作成**です。

塾講師が週次・月次で作成するこれらのドキュメントは、一般的に以下のような頻度で発生します。

- **生徒評価シート（定期）**：月1〜2回 × 在籍生徒数（平均30〜100名）
- **進路指導報告書**：受験シーズン（10月〜2月）に集中して大量発生
- **保護者向け面談準備資料**：学期ごと・面談前後に毎回

講師一人が月に費やす書類作成時間は、ヒアリングベースで**20〜40時間**。時給換算すれば、授業の機会損失や残業コストは月3〜8万円相当になります。しかも多くの塾では「テンプレートすら整備されていない」のが実態です。

一方でAI技術の進歩により、**生徒の成績データや授業記録をインプットするだけで、自然言語の評価文を自動生成する**ことが2023〜2024年にかけて現実的になりました。Claude 3.5 SonnetやGPT-4oは、教育分野の専門的な語彙・表現・トーン調整において非常に高い精度を発揮します。

さらに重要なのが「担い手不足」です。塾経営者や教室長は教育のプロですが、ITツールの導入・設定は完全に専門外。「AIを使って書類を効率化したい」という需要は実在しているのに、**誰も仕組みを作ってくれる人がいない**という状況が全国5万塾で起きています。

副業として参入するタイミングとしては、今がまさにブルーオーシャン期です。競合が少なく、単価交渉がしやすく、一度導入すれば継続契約に繋がりやすい業種です。

## AIで自動化できる部分の設計

このビジネスの核心は「インプットデータを整えれば、ドキュメント生成の80%をAIが担える」という点です。ワークフロー全体を設計すると以下になります。

### ワークフロー全体図

```
【インプット】
塾からのデータ入力（Googleフォーム or Notion）
↓
【データ整形】
Make.com（旧Integromat）でデータを整形・分類
↓
【AI生成】
Claude API（claude-3-5-sonnet）にプロンプトを送信
↓
【ドキュメント出力】
Google Docsに自動書き込み → PDF変換
↓
【納品】
塾側のGoogleドライブフォルダに自動格納 or メール送信
```

### 各ツールの役割

**① Googleフォーム（データ収集）**
塾講師が入力するフォームを設計します。項目例は以下の通り。
- 生徒名・学年・担当講師名
- 今月の出席状況（数値）
- 主要科目の点数（前回比）
- 授業中の様子（テキストフィールド：50〜200字程度で講師が入力）
- 課題の提出状況（プルダウン）
- 保護者への特記事項の有無（Yes/No）
- 志望校・進路の変動情報（受験生のみ）

このフォームへの入力負担を「授業後5分以内」に抑えることが、塾側の継続利用を左右します。講師が書いた「メモ書き程度の情報」をAIが「丁寧な評価文」に変換するという体験設計が重要です。

**② Make.com（ワークフロー自動化）**
Googleフォームの送信をトリガーに、データをJSON形式に整形してClaude APIに渡します。月額無料プランで月1,000オペレーション、有料（$9/月）で1万オペレーション処理できます。5〜10社の塾クライアントを持つなら有料プランで十分対応可能です。

**③ Claude API（文章生成）**
Anthropicの`claude-3-5-sonnet-20241022`モデルを使用します。APIキーはAnthropic公式（console.anthropic.com）から取得。1,000トークンあたり約$0.003（インプット）/$0.015（アウトプット）と非常に安価です。生徒1人の評価シート生成コストは$0.01〜0.03程度。

**④ Google Docs API + Zapier（ドキュメント生成・配布）**
生成されたテキストをGoogleドキュメントのテンプレートに自動挿入し、塾専用のドライブフォルダに格納します。Zapierの無料プランでも「フォーム送信 → ドキュメント作成」の基本フローは動かせます。

## 実際のツール設定（ステップバイステップ）

### STEP 1：Claude APIキーの取得とプロンプト設計

1. `console.anthropic.com` にアクセスしてアカウント作成
2. 「API Keys」から新しいキーを発行し、安全な場所に保管
3. 以下のシステムプロンプトを作成（これがサービス品質の核心）

```
あなたは学習塾の熟練した教務担当者です。
講師が入力した生徒情報をもとに、
保護者への報告に適した丁寧かつ温かみのある評価文を作成してください。

【出力形式】
- 冒頭：今月の全体的な取り組みへの評価（2〜3文）
- 学習面：科目別の成長・課題（箇条書き3〜4項目）
- 生活面・授業態度：前向きな表現を使用すること
- 来月への目標・メッセージ（保護者へ向けた1段落）
- 文体：ですます調、過度に専門的な語彙は避ける
```

4. ユーザーメッセージには `{生徒名}、{学年}、{今月の様子}、{成績データ}` を動的に差し込む変数構造にする

### STEP 2：Googleフォームの設計

1. Googleフォームで新規フォームを作成
2. 「回答をスプレッドシートに収集」をオンにする
3. 各塾専用のフォームURL（共有しない）を発行し、講師に案内
4. フォームのセクションを「基本情報」「学習記録」「進路情報（受験生のみ）」の3ブロックに分ける

### STEP 3：Make.comのシナリオ設定

1. make.com でアカウント作成（無料）
2. 「Create a new scenario」をクリック
3. トリガー：`Google Sheets > Watch New Rows`（フォーム回答シートを指定）
4. モジュール追加：`HTTP > Make an API Call`
   - URL: `https://api.anthropic.com/v1/messages`
   - Method: POST
   - Headers: `x-api-key: {your_key}`, `anthropic-version: 2023-06-01`, `content-type: application/json`
   - Body: 上記のシステムプロンプト＋フォームデータを動的にマッピング
5. レスポンス（生成テキスト）を次のモジュールに渡す
6. `Google Docs > Create a Document from Template` で整形済みドキュメントを生成
7. `Google Drive > Move a File` で塾専用フォルダに格納
8. 任意で `Gmail > Send an Email` で完了通知を塾側に送信

### STEP 4：テスト運用とプロンプト調整

1. ダミーデータでフォーム送信し、出力を確認
2. 生成文のトーン・語彙・情報の漏れを手動でチェック
3. 問題があればシステムプロンプトを修正（ここが最も重要な職人作業）
4. 塾ごとに「保護者層の年齢帯」「塾のカラー（厳しめ/温かめ）」でプロンプトを微調整

## 自動化できない部分（ここが重要）

AIが文章を生成できても、**人間が価値を出し続けなければならない領域**があります。これがコモディティ化を防ぐ最大の差別化ポイントです。

**① 塾固有の文化・言語への翻訳**
各塾には独自のカルチャーがあります。「この塾は生徒に対して厳しく接する方針」「保護者へは必ず前向きな言葉で締める」「講師の名前は全て敬称略で記載する」など、ドキュメントに反映すべきローカルルールはヒアリングしなければわかりません。

**② 初期ヒアリングと関係構築**
塾長・教室長との最初のミーティングで「どんな評価シートを作りたいか」「保護者はどんな層か」「過去の報告書の問題点は何か」を丁寧に聞き出す作業は、AIには絶対にできません。この時間こそが継続契約の基盤を作ります。

**③ AIアウトプットの品質チェック**
生成された文章が「事実と異なるポジティブ表現になっていないか」「学校名・志望校名の誤記がないか」「特定の生徒への配慮事項が反映されているか」を確認するのは人間の責任です。特に**進路指導報告書は情報の誤りが受験に直結する**ため、チェック工数を削れません。

**④ 緊急対応・イレギュラーケース**
不登校傾向の生徒、保護者からのクレームを受けた後の報告書、急な転塾・退塾時の対応など、通常フローに乗らないケースは人間が個別対応する必要があります。

**⑤ 定期的なプロンプトのアップデート**
教育トレンド（新学習指導要領対応など）や塾側の方針変更に合わせてプロンプトを見直す作業も、継続的な価値提供の一部です。

## 参入のボトルネック

正直に書きます。以下の障壁を把握してから参入してください。

**① 最初のクライアント獲得が最大の壁**
このサービスはまだ「認知されていない需要」です。塾経営者は「評価シートをAIで作れる」という発想を持っていないため、営業時には「課題の認識」から始める必要があります。飛び込み営業やコールドメールへの耐性がない人には辛い時期があります。

**② 教育業界への信頼構築**
塾経営者は「個人情報（生徒情報）」を扱うことへの慎重さが高い。GDPR・個人情報保護法への対応として、データの取り扱いポリシー・利用規約を整備しておく必要があります。弁護士ドットコムのひな形サービス（月額3,300円〜）を活用して最低限の契約書を用意することを強く推奨します。

**③ プロンプトエンジニアリングのスキル習得**
「それっぽい文章」と「実際に使える文章」の差はプロンプト設計で決まります。最初の1〜2週間はひたすらテスト生成と改善に時間を使う必要があります。この投資を惜しむと、クライアントが離脱します。

**④ 繁閑の差が激しい**
受験シーズン（11〜2月）は案件が集中し、4〜7月は比較的閑散とします。月額継続契約をうまく設計しないと収益が安定しません。オフシーズンには「入塾案内文書の作成」「年間シラバスのテンプレ化」など関連サービスで単価を補う設計が必要です。

**⑤ Make.comなど自動化ツールの初期習得コスト**
完全な初心者であれば、Make.comの基本操作習得に5〜10時間は必要です。ただしYouTubeの日本語解説動画が充実しているため、独学で乗り越えられるレベルです。

## 最初の一歩

今日から動ける3ステップに絞ります。難しく考えず、まず手を動かしてください。

### STEP 1：Claude APIのアカウントとテスト生成（今日中に完了）

1. `console.anthropic.com` にアクセスしてアカウント作成（無料）
2. クレジットカードを登録し、$5分のクレジットをチャージ（最小単位）
3. 「Workbench」機能を使い、以下のプロンプトでテスト生成を行う

```
システム：あなたは学習塾の教務担当です。生徒の評価シートを保護者向けに作成してください。
ユーザー：中学2年生の田中太郎。今月の数学：65点（前回比+8点）、英語：72点（前回比-3点）。
授業態度：積極的に質問してくれる。宿題提出率：80%。担当講師のコメント：本人のやる気は高いが英語の語彙力強化が必要。
```

4. 生成された文章のクオリティを確認し、プロンプトを5〜10回改善する

この体験が「サービスの核心」への理解を一気に深めます。

### STEP 2：Make.comの無料アカウント作成とサンプルフロー構築（3日以内）

1. `make.com` でアカウント作成（クレジットカード不要）
2. 「Templates」から `Google Forms > Google Docs` のテンプレートを探して複製
3. 自分でダミーのGoogleフォームを作り、実際にデータが流れるかテスト
4. HTTPモジュールを追加してClaude APIと繋ぐ（Make.comのYouTube公式チャンネル：`youtube.com/@Make` で「HTTP module tutorial」で検索）

### STEP 3：最初の1社へのアプローチ（1週間以内）

**ターゲット選定の優先順位**：
- まず自分の知り合い・家族が通っている/経営している塾

- ランサーズ（`lancers.jp`）でも同様に「教育 事務 代行」で検索

**初回提案のポイント**：
「無料で1ヶ月の生徒5名分の評価シートを作成します。気に入ったら継続契約を検討いただければ」という無料トライアルオファーが最も刺さります。完成サンプルを1枚だけ作って見せると、成約率が大幅に上がります。

補足として、教育業界特化のコミュニティ「じゅくちょー（jukucho.jp）」のようなフォーラムや、塾経営者が集まるFacebookグループへの参加も情報収集・営業の入り口として有効です。
