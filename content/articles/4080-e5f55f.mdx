# 人材紹介・採用支援向け 職務経歴書・面接評価レポート作成代行

## AIビジネス解剖室 ｜ 採用支援 × AI自動化の完全ガイド

---

## なぜ今なのか

日本の人材紹介市場は2023年度に約3兆円規模に達し、登録人材紹介会社数は28,000社を超えた（厚生労働省「職業紹介事業報告書」2023年度版）。求人倍率が慢性的に高止まりするなか、転職支援の需要は構造的に拡大し続けている。

しかし、現場では深刻な「書類作成の工数問題」が発生している。

中小の人材紹介会社やフリーランスのキャリアアドバイザーは、1人あたり月に30〜80名の候補者を担当する。1名につき職務経歴書の添削・ブラッシュアップに平均45〜90分、企業への推薦状（求職者プロフィールシート）作成に30〜60分かかる。単純計算で、月に60時間以上が「書類作成」だけに消えていく。しかも、この作業は単調で高度な判断を要せず、アドバイザー本来の業務（候補者面談・企業開拓）の時間を圧迫している。

面接評価レポートも同様だ。採用企業の人事担当者が面接後に作成する「評価サマリー」や「合否判断ドキュメント」は、面接1件あたり30〜60分を要する。採用ペースが上がるほど工数が爆増し、「誰かに外注したい」という声が絶えない。

ここにAI活用の巨大な商機がある。

GPT-4やClaudeの登場により、構造化されたインプット情報（職歴・スキル・希望条件）から質の高い文書を生成する精度が飛躍的に向上した。2024年時点でClaude 3.5 Sonnetは長文の文脈理解・日本語の敬語ニュアンス・ビジネス文書作成において人間のアシスタントレベルに匹敵する出力品質を持つ。つまり「AI単体でドラフトを作り、人間が最終チェックする」という分業モデルが現実的に機能する段階に来ている。

さらに重要なのが「担い手不足」だ。現役の人材紹介会社員がこのサービスを外注先として探しても、「採用ドメインの知識がある＋AIを使いこなせる」人材は市場にほとんど存在しない。需要過多・供給不足のブルーオーシャンが今この瞬間も広がっている。

## AIで自動化できる部分の設計

このビジネスの核心は「構造化インプット → AI生成 → 人間レビュー → 納品」というパイプラインを構築することにある。以下に具体的なワークフローを示す。

### ワークフロー全体像

```
候補者情報収集（Googleフォーム）
        ↓
自動トリガー（Make.com / n8n）
        ↓
Claude API / GPT-4 APIで文書生成
        ↓
Google Docsに自動保存
        ↓
Slackまたはメールで通知
        ↓
人間がレビュー・修正（15〜20分）
        ↓
クライアントへ納品
```

### 使用ツールの役割分担

**① Googleフォーム（無料）**
候補者の基本情報を収集するフロントエンド。設問例：氏名・年齢・現職・職歴サマリー（5年分）・保有スキル・資格・希望職種・強みと弱み・転職理由。フォームへの入力は候補者本人またはアドバイザーが行う。

**② Make.com（旧Integromat）**
Googleフォームへの回答を検知し、Claude APIへのリクエストを自動送信する。月9ドルのBasicプランで十分動作する。フォームの回答データをJSON形式に整形し、APIリクエストのbodyとして渡す処理を担う。

**③ Claude API（Anthropic）**
メインの生成エンジン。プロンプトに「人材紹介会社向け職務経歴書のブラッシュアップ専門家」というシステムプロンプトを設定し、候補者情報から①職務経歴書ドラフト②推薦コメント文③強みの要約（200字）の3点セットを一括生成させる。APIコストはClaude 3.5 Sonnetで入力100万トークンあたり$3、出力100万トークンあたり$15。1件あたりの処理コストは約$0.05〜$0.10と極めて安価。

**④ Google Docs API**
生成されたテキストを指定のGoogleドキュメントテンプレートに自動挿入。クライアントごとのフォルダ構成を事前に作っておけば、納品ドキュメントが自動で所定の場所に格納される。

**⑤ Slack Webhook（または Gmail）**
「〇〇さんの職務経歴書ドラフトが完成しました」という通知を自分のSlackチャンネルに自動送信。レビュー忘れを防ぐ。

## 実際のツール設定（ステップバイステップ）

### STEP 1：Googleフォームを作成する

1. Googleフォーム（forms.google.com）で新規フォームを作成
2. 以下の設問を追加する：
   - 候補者氏名（短文）
   - 現在の職種・役職（短文）
   - 直近5年間の職務経歴（段落テキスト）
   - 保有スキル・資格（段落テキスト）
   - 転職理由・希望条件（段落テキスト）
   - 自己PR・強みと弱み（段落テキスト）
   - 対象企業の業種・求人JD概要（段落テキスト）
3. 「回答」タブ → 「Googleスプレッドシートにリンク」を設定する

### STEP 2：Make.comでワークフローを構築する

1. make.com にサインアップ（無料プランで試作可能）
2. 「新しいシナリオを作成」をクリック
3. **トリガーモジュール**：「Google Sheets」→「Watch New Rows」を選択し、STEP 1で作ったスプレッドシートを指定。ポーリング間隔は15分に設定
4. **HTTP モジュール**を追加：「HTTP」→「Make an API Key Auth Request」を選択
   - URL：`https://api.anthropic.com/v1/messages`
   - Method：POST
   - Headers：`x-api-key: （AnthropicのAPIキー）`、`anthropic-version: 2023-06-01`、`content-type: application/json`
   - Body（JSON形式）：
     ```json
     {
       "model": "claude-3-5-sonnet-20241022",
       "max_tokens": 4096,
       "system": "あなたは10年以上のキャリアを持つ人材紹介会社のトップコンサルタントです。候補者情報をもとに、①採用企業に響く職務経歴書ドラフト、②推薦コメント（300字）、③強みの要約（200字）を出力してください。出力はJSON形式で。",
       "messages": [
         {
           "role": "user",
           "content": "【候補者情報】\n氏名：{{1.氏名}}\n現職：{{1.現職}}\n職歴：{{1.職歴}}\nスキル：{{1.スキル}}\n転職理由：{{1.転職理由}}\n自己PR：{{1.自己PR}}\n対象求人：{{1.求人JD}}"
         }
       ]
     }
     ```
5. **Google Docs モジュール**を追加：「Google Docs」→「Create a Document from a Template」を選択
   - 事前に「職務経歴書テンプレート」をGoogleドキュメントで作成し、`{{職務経歴書}}` などのプレースホルダーを設定しておく
   - Make.comのこのモジュールで、Claudeの出力をプレースホルダーにマッピングする
6. **Slack モジュール**を追加：「Slack」→「Create a Message」で自分のチャンネルに完了通知を送信
7. シナリオを「オン」にして保存

### STEP 3：Claude APIキーを取得する

1. console.anthropic.com にアクセスしてアカウント作成
2. 「API Keys」→「Create Key」でAPIキーを発行
3. 初回クレジット（$5相当）が付与されるので、テスト運用に使用
4. 本番運用はクレジットカードを登録してプリペイド形式でチャージ

### STEP 4：品質チェックリストを作成する

自動生成されたドラフトを人間がレビューする際の確認リストをNotionまたはGoogleドキュメントで作成しておく。チェック項目例：
- 数字・実績データの整合性（AIが数値を盛っていないか）
- 業界用語の適切な使用
- 敬語・文体のトーン統一
- 対象求人とのマッチング度合い

## 自動化できない部分（ここが重要）

AIが高品質なドラフトを生成できても、以下の部分は必ず人間が担う必要がある。これこそがコモディティ化を防ぐ「本質的な差別化ポイント」だ。

**① 候補者との信頼関係の構築と情報引き出し**
フォームに書かれた内容だけでは実績の「深み」は見えない。「その成果を出せた理由は何ですか？」「チームの何人をまとめていたのですか？」という深掘りインタビューは人間にしかできない。そこで得た情報を追加でAIに渡すと、ドラフトの質が格段に上がる。

**② 特定の企業文化・採用担当者のクセへの対応**
「A社の採用担当者は数値実績を重視する」「B社はリーダーシップエピソードを必ず聞く」という属人的な知識は、データに存在しない。この「読み」の反映はキャリアアドバイザーの経験知が必要。

**③ 倫理的・法的リスクの判断**
AIは候補者を良く見せようとして「実績を誇張した表現」を生成する場合がある。経歴詐称リスク・虚偽記載に該当する表現を除去する判断は人間が行わなければならない。

**④ クライアントへのコンサルティング提案**
「この候補者はC社よりもD社のほうが内定確率が高い」という市場感覚に基づく提案や、職務経歴書の方向性についてのアドバイスは、AIが出力した文書の品質チェックとは別次元の価値提供だ。

**⑤ 緊急対応・イレギュラー対応**
「明日の面接に間に合わせてほしい」「フォーマットを企業指定のものに合わせてほしい」という個別要望への対応は自動化ワークフローの範囲外。ここへの対応力が高い評価・リピートにつながる。

## 参入のボトルネック

正直に書く。この副業にはいくつかの現実的な障壁がある。

**① 採用ドメイン知識の習得コスト**
職務経歴書の「何が良いか」を判断するには、業種別の採用基準・ATS（採用管理システム）のスクリーニングロジック・現在の労働市場動向の最低限の理解が必要。これは1〜2週間の独学では追いつかない。人材業界での就業経験がない場合、まずクラウドワークスで低単価案件を受けながら実地で学ぶ期間（1〜3ヶ月）が必要。

**② 初期クライアント獲得の難しさ**
「実績ゼロ」の状態で人材紹介会社に営業しても信用されにくい。最初の2〜3件は採算度外視（無料〜格安）でポートフォリオを作る覚悟が必要。

**③ APIの技術的ハードル**
Make.comとClaude APIの連携は、プログラミング経験がない人には最初ハードルに感じる。ただし、Make.comはノーコードツールであり、YouTubeに日本語解説動画も増えている。2〜3日かければ基本ワークフローは構築できる。

**④ 情報漏洩リスクへの配慮**
候補者の個人情報・職歴を外部のAI APIに送信することへの倫理的・法的リスクがある。AnthropicのAPIは商用利用時にデータ学習に使用しないポリシーを持つが、クライアントにはこの点を説明し、候補者の同意取得フローを確認する必要がある。個人情報保護法の観点から、「外部AIサービスへのデータ提供」について利用規約・同意書を整備することを強く推奨する。

**⑤ 差別化の陳腐化リスク**
1〜2年後には同様のサービスが増え、単価競争が起きる可能性がある。早期参入して「採用支援AIの専門家」としてのブランドを確立することが、長期的な収益確保のカギになる。

## 最初の一歩

今日この記事を読んだあなたが、今夜から始められる3ステップを示す。

### STEP 1：無料でワークフローのプロトタイプを作る（今日〜3日以内）

1. **Make.com に無料登録**（https://www.make.com/en/register）
   → 無料プランでシナリオ2件・月1,000オペレーション利用可能
2. **Anthropic コンソールに登録してAPIキーを取得**（https://console.anthropic.com）
   → 初回$5クレジット付き。約50〜100件分のテスト処理が可能
3. Googleフォームを作成し、本記事のSTEP 2で示したMake.comシナリオを構築。自分自身の職歴をサンプルデータとして流し込み、ドラフトが生成されるかを確認する

### STEP 2：ポートフォリオを作って最初の案件を取る（3日〜2週間以内）

1. **クラウドワークス（https://crowdworks.jp）に登録**し、「職務経歴書作成」カテゴリで既存案件に応募。最初の1〜3件は3,000〜5,000円の低単価でも受ける
2. 納品した案件の「ビフォー・アフター比較」（候補者の元の文章とAI×自分で仕上げた文章の対比）をポートフォリオとして作成
3. 並行して**ランサーズ（https://www.lancers.jp）**にもプロフィールを登録。「採用支援」「職務経歴書」のタグを設定

### STEP 3：人材紹介会社への直接営業でリカーリング収益を作る（2週間〜1ヶ月以内）

1. **Googleマップ・求人ボックス**で「人材紹介会社 ＋ 地域名（または業種名）」で検索し、中小規模（従業員5〜30名）の会社をリストアップ（50社目標）
2. LinkedInまたはメールで「採用書類作成をAIでサポートするサービスを月3万円から提供しています。まず1件無料でお試しください」という内容の営業文を送付
3. 初回トライアルを経て月次契約に移行。契約1社目が取れた時点で、本記事のワークフローを本番稼働させる
