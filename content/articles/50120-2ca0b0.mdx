# 不動産賃貸物件の家賃査定・相場分析レポート作成代行

## なぜ今なのか

日本の賃貸住宅市場は約1,870万戸（総務省「住宅・土地統計調査」2023年）という巨大な規模を誇る。しかし、この市場を動かしている不動産オーナーの多くは、適正な家賃設定に慢性的な悩みを抱えている。国土交通省のデータによれば、個人で賃貸物件を所有するオーナーは全体の約60%以上を占め、そのほとんどが管理会社任せの家賃設定を行っている。言い換えれば「自分の資産の適正価格を自分で判断できていない」オーナーが大量に存在する。

商機の背景はさらに具体的だ。SUUMOやアットホームなどの不動産ポータルサイトには毎日膨大な物件情報が更新されており、スクレイピング・API活用によってリアルタイムの相場データを取得できる環境が整ってきた。一方で、従来の家賃査定は「管理会社の担当者が感覚的に決める」「REINSのデータを手動で調べる」という非効率なプロセスに頼っていた。

AIの進歩がここに風穴を開けた。GPT-4やClaudeは不動産データの解析・自然言語によるレポート生成において2023〜2024年に劇的に精度が向上した。以前は専門家が数時間かけて作成していた相場分析レポートが、適切なプロンプト設計とデータ収集の自動化によって30分以内で生成できる時代になっている。

さらに注目すべきは担い手不足だ。クラウドワークスやランサーズを検索すると「不動産相場調査」「賃貸物件の競合分析」というカテゴリの案件は常時存在するが、「不動産知識×データ分析×AIレポート生成」を組み合わせられる人材はほぼいない。需要が供給を大きく上回っているブルーオーシャンの典型例である。

## AIで自動化できる部分の設計

この業務の自動化は「データ収集→分析→レポート生成→納品」という4段階で設計する。

**【データ収集フェーズ】**
Make.com（旧Integromat）を使い、SUUMOやホームズの賃貸情報をWeb Scraping Moduleで定期収集する。具体的には対象エリア・間取り・築年数の条件をパラメータ化し、1日1回自動でデータを更新してGoogleスプレッドシートに蓄積する。Apify（https://apify.com）のSUUMO専用スクレイパーActorを使えばコーディング不要で物件データのJSON取得が可能だ。

**【データクレンジング・分析フェーズ】**
収集したローデータをPythonスクリプト（Google Colab上で実行）で前処理する。間取りコードの統一、面積あたり家賃（円/㎡）の算出、外れ値除去を自動化。これをCSV形式でClaude APIまたはGPT-4 APIに渡す。

**【レポート生成フェーズ】**
Claude API（claude-3-5-sonnet）に対して以下の構造化プロンプトを送信する：

```
あなたは不動産市場の専門アナリストです。
以下のデータに基づき、{物件住所}周辺の賃貸相場分析レポートを作成してください。

[データ]: {CSVデータ}
[対象物件スペック]: 築{X}年、{間取り}、{面積}㎡、{駅徒歩}分
[レポート形式]: エグゼクティブサマリー/相場分布/競合比較/推奨家賃帯/根拠説明
```

このプロンプト設計により、A4換算3〜5ページの構造化レポートが自動生成される。

**【納品フェーズ】**
n8nのワークフローで、生成されたレポートをGoogle DocsまたはPDF形式に自動変換し、クライアントのメールアドレスに自動送信する。

## 実際のツール設定（ステップバイステップ）

**【STEP 1: Apifyでデータ収集の設定】**
1. https://apify.com にアクセスし、無料アカウントを作成（月$49のStarterプランで十分）
2. Storeから「SUUMO Scraper」または「Real Estate Japan Scraper」を検索してActorを選択
3. Input設定で「prefecture: 東京都」「ward: 新宿区」「layout: 1LDK」「maxItems: 100」を入力
4. Schedulerで「毎日午前6時に実行」を設定
5. OutputをWebhookでMake.comに送信するよう設定（ActorのIntegration設定から）

**【STEP 2: Make.comでデータパイプライン構築】**
1. https://make.com にアクセスし、無料アカウント作成
2. 新規シナリオを作成し、「Webhooks > Custom Webhook」をトリガーに設定
3. ApifyからのJSONデータを受け取る「Parse JSON」モジュールを追加
4. 「Google Sheets > Add Row」モジュールを追加し、物件名・家賃・面積・築年数・駅徒歩分数のフィールドをマッピング
5. シナリオを保存し、Apify側のWebhook URLに貼り付けて連携完了

**【STEP 3: Claude APIでレポート自動生成】**
1. https://console.anthropic.com でAPIキーを取得（$5チャージで約50〜100件分のレポート生成が可能）
2. Google Colab（https://colab.research.google.com）で以下を実装：
   - `gspread`ライブラリでスプレッドシートからデータ取得
   - pandas で統計処理（中央値・四分位数・最頻値算出）
   - anthropic ライブラリで`claude-3-5-sonnet-20241022`モデルにプロンプト送信
   - レスポンスをMarkdown形式で保存
3. `anthropic.messages.create()`のmax_tokensを4000に設定し、詳細レポートを要求

**【STEP 4: n8nで納品自動化】**
1. https://n8n.io でクラウド版アカウント作成（月€20のStarterプラン）
2. 「Google Drive」ノードでMarkdownをGoogleドキュメントに変換
3. 「Gmail」または「SendGrid」ノードでPDFをクライアントにメール送信
4. 「Notion」ノードで案件管理DBに完了ステータスを自動記録
5. 全体のワークフローを「Webhook受信→レポート生成→PDF変換→メール送信→DB更新」で1トリガーに統合

## 自動化できない部分（ここが重要）

AIによる自動化が可能な部分が多い一方で、以下の要素は人間が必ず担う必要がある。ここが競合との差別化ポイントであり、あなたの付加価値になる。

**① クライアントヒアリングと要件定義**
「リノベーションを予定している」「入居者ターゲットを外国人に変えたい」「売却を視野に入れている」など、オーナーの個別事情はAIには読み取れない。初回のヒアリングで背景情報を引き出し、それをプロンプトに反映させるのは人間の仕事だ。

**② データの品質チェックと異常値判断**
スクレイピングデータには「礼金0・フリーレント3ヶ月付き」などの条件が数字に反映されていない場合がある。「なぜこの物件だけ極端に安いのか」を判断するには物件情報の行間を読む能力が必要で、これは現時点でAIが苦手とする領域だ。

**③ 地域特性のコンテキスト補完**
「この駅は再開発計画があり2026年に大型商業施設が開業予定」「この通りは夜間騒音が問題で空室率が高い」といったローカル情報は公開データに存在しないことが多い。地域の定性情報を分析に組み込むのは人間の仕事だ。

**④ クライアントへの提案・説明**
レポートの内容をオーナーに対して説得力を持って説明し、「では月額8万5千円で設定しましょう」という意思決定サポートは人間にしかできない。特に高齢のオーナーへの対応では信頼関係の構築が重要になる。

**⑤ 法的・倫理的チェック**
家賃設定に関するカルテル規制（不当廉売・談合への注意）、個人情報の取り扱い、スクレイピングの利用規約遵守などの確認は人間が行う必要がある。

## 参入のボトルネック

正直に書く。この分野にはいくつかの現実的な障壁がある。

**① 不動産知識の学習コスト**
家賃査定には「利回り計算」「キャップレート」「周辺施設の評価方法」など基礎知識が必要だ。まったくの未経験者は最低でも宅建の市販テキスト1冊と実際の物件情報を数百件読む学習期間（1〜2ヶ月）が必要になる。

**② スクレイピングの法的グレーゾーン**
SUUMOやホームズなどの利用規約は「自動的なデータ収集を禁止」と明記している場合が多い。国土交通省が提供するREINS（不動産流通標準情報システム）は宅建業者にしかアクセス権がない。法的リスクを避けるには、オーナー自身が提供するデータの分析に特化するか、公開されているe-Statや地価公示データの活用に留める必要がある。

**③ 初期クライアント獲得の難しさ**
「家賃査定AI」への信頼性は最初は低い。実績ゼロの状態では案件獲得が困難なため、最初の2〜3件は大幅に割引（無料〜5,000円）して実績とレビューを積む必要がある。

**④ データの鮮度管理**
賃貸相場は季節・経済状況で変動する。過去データに基づくレポートが「古い情報だ」とクレームになるリスクがある。データ収集の頻度管理とレポートへの「データ取得日の明記」が必須だ。

**⑤ Claude・GPT-4 APIのコスト管理**
claude-3-5-sonnetでA4×5ページ相当のレポートを生成すると1件あたり約$0.10〜0.30（約15〜45円）かかる。月100件処理しても数千円程度だが、プロンプトが長大化するとコストが跳ね上がるため、トークン数の最適化が必要だ。
