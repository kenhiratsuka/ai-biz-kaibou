# 不動産賃貸管理向け入居者対応・督促メール自動生成代行｜AIビジネス解剖室

---

## なぜ今なのか

日本の賃貸管理市場は巨大だ。国土交通省の統計によれば、全国の賃貸住宅戸数は約1,900万戸を超え、これを管理する不動産管理会社は全国に約1万社以上存在する。そのうち管理戸数が1,000戸未満の中小管理会社が全体の約7割を占めるという構造がある。

問題は人手だ。賃貸管理業務のうち、入居者からの問い合わせ対応・家賃督促・更新案内・退去手続き案内といったコミュニケーション業務は、担当者1人が数百〜1,000戸を抱えながらこなすのが実態だ。家賃滞納率は全国平均で約3〜5%程度とされており、仮に管理戸数500戸の会社であれば毎月15〜25件の滞納案件に対応しなければならない。それぞれに督促の初回連絡・2回目・法的通知前の最終警告、さらには入居者の状況に応じた個別文面の調整が必要になる。これを手書きやコピペで対応している担当者が今もほとんどだ。

ここにAIの商機がある。ChatGPTやClaudeの登場以降、「メール文章を自動生成する」というユースケースは技術的にはほぼ解決済みだが、**不動産管理という特定業種に特化したプロンプト設計・ワークフロー構築・継続運用を丸ごと代行できるプレイヤーがまだほとんどいない**。

国内のクラウドワークス・ランサーズを見ても、「AIライティング代行」の案件は多数存在するものの、「賃貸管理会社向けのAI督促メール自動化構築」というニッチに絞った出品は2024年末時点でほぼゼロに近い。海外のFiverrでは"property management automation"というカテゴリでZapier・Make連携の自動化サービスが$200〜$800/プロジェクトで取引されているが、日本語対応・国内法規制対応ができる出品者はほぼ皆無だ。これはブルーオーシャンと言っていい。

## AIで自動化できる部分の設計

自動化の全体像は「情報収集 → 文章生成 → 配信 → 記録」という4ステップで設計できる。

**ステップ1：トリガー設計（情報収集）**
管理会社が使う物件管理ソフト（Googleスプレッドシートで代用している会社も多い）に、入居者の氏名・部屋番号・滞納月数・過去の対応履歴・督促ステージ（初回/2回目/最終）などの情報が格納されている。これをn8nまたはMake.comが定期的に読み込み、「今日督促が必要な入居者リスト」を自動抽出するようトリガーを設定する。

**ステップ2：プロンプト設計（Claude APIで文章生成）**
抽出した入居者情報をClaude APIに渡し、状況に応じた文面を生成させる。ここがこのビジネスの核心だ。プロンプトには以下の情報を動的に埋め込む。

- 入居者氏名・部屋番号
- 滞納額・滞納期間
- 督促ステージ（初回は柔らかいトーン、3回目は法的措置示唆）
- 過去のやり取りの要約
- 管理会社の社名・担当者名

Claude APIのSystem Promptには「あなたは日本の賃貸管理会社の担当者として丁寧かつ毅然とした文章を書くアシスタントです。借地借家法・消費者保護法の観点から不当な表現は使わず、法的に問題のないメール文面を生成してください」という指示を設定しておく。これにより、ハラスメント的な督促文（「今すぐ払え」等）を自動的に排除できる。

**ステップ3：ヒューマンレビューキュー（半自動化）**
生成された文面をそのまま送信するのではなく、一度Googleスプレッドシートまたはメール下書きフォルダに格納し、担当者が30秒〜1分でレビューできる状態にする。この「送信前確認」ステップを必ず挟むことが後述する法的リスク管理の観点から重要だ。

**ステップ4：配信・記録**
承認後、n8nのGmailノードまたはSendGrid APIで配信。配信ログをスプレッドシートに自動記録し、次回の督促判定に使用する。

## 実際のツール設定（ステップバイステップ）

以下はn8n（セルフホストまたはn8n Cloud）とClaude APIを組み合わせた構成の具体的な手順だ。

**1. n8nのインストールと初期設定**
n8n Cloudのフリートライアル（https://n8n.io）に登録する。セルフホストする場合はDockerコマンド`docker run -it --rm --name n8n -p 5678:5678 n8nio/n8n`で即起動できる。

**2. Googleスプレッドシートノードの設定**
n8nのワークフロー画面で「Add Node」→「Google Sheets」を選択。OAuth2でGoogleアカウントを連携し、管理会社が共有するスプレッドシートのURLとシート名を入力する。「督促対象」フラグがTRUEの行だけをフィルタするFilter条件を設定する（例：`{{$json["督促フラグ"]}} === "TRUE"`）。

**3. Scheduleトリガーの設定**
毎営業日の朝9時にワークフローを起動するスケジュールを設定。Cronノードで`0 9 * * 1-5`と入力する。

**4. Claude API（HTTP Requestノード）の設定**
n8nの「HTTP Request」ノードを追加し、以下を設定する。
- URL：`https://api.anthropic.com/v1/messages`
- Method：POST
- Header：`x-api-key: {{YOUR_CLAUDE_API_KEY}}`、`anthropic-version: 2023-06-01`
- Body（JSON）：
```json
{
  "model": "claude-opus-4-5",
  "max_tokens": 1024,
  "system": "あなたは日本の賃貸管理会社の担当アシスタントです...",
  "messages": [{"role": "user", "content": "{{動的プロンプト}}"}]
}
```
動的プロンプト部分には`{{$json["氏名"]}}様、{{$json["滞納月数"]}}ヶ月分の家賃{{$json["滞納額"]}}円が...`のようにスプレッドシートのデータを埋め込む。

**5. Googleスプレッドシートへの出力（レビューキュー）**
生成された文面を別シート「督促メール下書き」に書き込む。列構成は「日付・入居者名・部屋番号・生成文面・承認チェックボックス」とする。管理会社の担当者はこのシートを確認し、チェックを入れた行だけが次のステップに進む。

**6. Gmail送信ノードの設定**
n8nのGmailノードで、承認済み行をトリガーにメール送信。To欄には入居者のメールアドレス（スプレッドシートから取得）、本文には生成済みの文面を挿入。Cc欄に管理会社担当者を自動設定することで送信記録も残る。

**7. ログ記録**
送信完了後、スプレッドシートの「送信ログ」シートに日時・送信先・件名を自動記録。次回督促判定のための「前回送信日」フィールドも更新する。

## 自動化できない部分（ここが重要）

ここが人間の専門家としての価値が宿る場所だ。むしろこのパートを押さえているかどうかが、単なる「自動化ツール屋」と「コンサルタント」の差になる。

**① 法律的判断が必要なケースの仕分け**
滞納が3ヶ月を超えて保証会社への通知タイミングが来た場合、内容証明郵便が必要な段階になった場合、入居者が「支払う意思はある」と口頭連絡してきた場合など、AIが判断を誤ると後の法的手続きに影響が出る。この仕分けは人間が行う必要がある。

**② 管理会社との初期ヒアリングと設計**
どの物件管理ソフトを使っているか、督促の社内フローはどうなっているか、保証会社との契約形態は何かなど、個社ごとに異なる情報をヒアリングして設計するのは人間にしかできない。

**③ プロンプトの継続チューニング**
生成された文面が「硬すぎる」「柔らかすぎる」「この地域の言葉遣いに合わない」といったフィードバックを受けて調整するのは人間の感性と業種理解が必要だ。

**④ 入居者から返信が来た後の対応設計**
自動生成はあくまで「こちらからの発信」だ。入居者から「来月まで待ってほしい」「引っ越すことにした」という返信が来た場合の対応フローは個別対応になる。

**⑤ 個人情報管理の責任体制の整備**
入居者の氏名・連絡先・滞納情報は個人情報保護法上の要配慮情報に準じる扱いが必要だ。スプレッドシートのアクセス権限設定・APIキーの管理など、セキュリティ設計のアドバイスは人間の判断が必要だ。

## 収益モデルの試算

**料金設定の根拠**

Fiverrでは"property management email automation"カテゴリで類似サービスが$300〜$800/プロジェクト（初期構築費）＋$100〜$300/月（保守・改善費）で取引されている。Upworkでは自動化コンサルタントの時給は$50〜$150/hrが相場だ。

国内クラウドワークスでは業務自動化（RPA・API連携）の案件は50,000〜200,000円/件での発注が多く、ランサーズのシステム開発カテゴリでも同様の水準だ。

これらを参考に、国内の賃貸管理会社向けでは以下の料金設定が現実的だ。

| 項目 | 金額 |
|------|------|
| 初期構築費（ワークフロー設計・設定・テスト） | 80,000〜150,000円 |
| 月額保守・運用サポート費 | 20,000〜40,000円/月 |
| プロンプト改善・追加テンプレート | 10,000〜30,000円/回 |

**月収試算**

立ち上げ期（3〜6ヶ月目）の現実的な想定：

- 月額保守契約：3社 × 30,000円 ＝ **90,000円**
- 初期構築（新規月1件）：1社 × 100,000円 ＝ **100,000円**
- **合計月収：約19万円**

安定期（7〜12ヶ月目）の想定：

- 月額保守契約：8社 × 30,000円 ＝ **240,000円**
- 初期構築（月1〜2件）：1.5社 × 100,000円 ＝ **150,000円**
- **合計月収：約39万円**

**作業時間の見積もり**

初期構築：1社あたり15〜20時間（ヒアリング3時間・設計5時間・実装8時間・テスト4時間）。月額保守：1社あたり2〜3時間/月（ログ確認・フィードバック対応・プロンプト微調整）。8社保守なら月16〜24時間。副業として十分に両立できる水準だ。

**コスト**

Claude APIの費用はclaude-opus-4-5で入力$15/百万トークン・出力$75/百万トークン（2024年末時点）。1メール生成あたりのトークン数は約1,000〜2,000トークン程度なので、100件/月の生成でも$2〜5程度。n8n Cloudは月$20〜。実質的なツールコストは月5,000〜10,000円で収まる。

## 参入のボトルネック

**① 技術的ハードル：API連携経験がないと最初の壁が高い**
n8nやMake.comの基本操作はYouTubeで学べるが、HTTP RequestでAPIを叩く経験がない人には最初の数日は詰まる場面が出る。ただしChatGPTやClaudeに「n8nでClaude APIを使う方法を教えて」と聞けばコードごと答えてくれるので、プログラミング経験がなくても1〜2週間で動くものは作れる。

**② 営業ハードル：BtoB営業の経験が必要**
賃貸管理会社の担当者にリーチするには、飛び込み・テレアポ・紹介・クラウドソーシング受注のいずれかが必要だ。特に「AIが怖い・信頼できない」と感じる50代以上の管理会社オーナーへの説明スキルが求められる。デモ動画（Loomで録画した実際の自動化の様子）を用意すると説明が圧倒的に楽になる。

**③ 法的リスク：督促行為の適法性**
正当な債権回収行為（家賃督促）は合法だが、「脅迫的な文言」「深夜の連絡」「過度な連絡頻度」などは借地借家法・債権法・消費者契約法に抵触する可能性がある。自分が設計したシステムで問題が起きた場合の責任範囲を契約書で明確にしておくことが必須だ。弁護士ドットコムなどで契約書の雛形作成は数万円から依頼できる。

**④ 個人情報保護：プライバシーポリシーと委託契約**
入居者の個人情報を含むデータを扱うため、管理会社との間で個人情報の委託契約を締結する必要がある。また自分自身も個人情報取扱事業者として適切な管理体制が求められる。これを怠ると後々大きなトラブルになる。

**⑤ 解約リスク：管理会社が内製化する可能性**
一度システムを納品・構築すると、管理会社がそのまま自社で運用し始めて保守契約を解除するリスクがある。これを防ぐには「継続的なプロンプト改善・新テンプレート追加・法改正対応」など保守の価値を月次で可視化するレポート送付が有効だ。

## 最初の一歩

今日からできる具体的な3ステップを示す。

**STEP 1：ツールの無料アカウントを今日中に3つ作る**

まずn8n Cloudの無料トライアルに登録する（https://app.n8n.cloud/register）。次にAnthropicのコンソール（https://console.anthropic.com）でアカウントを作成し、$5分のAPIクレジットをチャージしてAPIキーを発行する。最後にMake.com（https://www.make.com/en/register）にも登録しておく（n8nと比較検討するため）。この3つの登録は合計30分で完了する。

**STEP 2：サンプルワークフローを1週間以内に完成させる**

「家賃1ヶ月滞納者向けの初回督促メール」を1通、Claudeで生成するだけのシンプルなワークフローをn8nで作る。スプレッドシートにダミーデータ（入居者名：山田太郎、滞納額：80,000円、部屋番号：203号室）を入れ、Claude APIから文面が返ってくるところまでを動かす。このデモ動画をLoom（https://www.loom.com）で画面録画しておく。これが営業ツールになる。

**STEP 3：最初のクライアントをクラウドワークスで獲得する**

クラウドワークス（https://crowdworks.jp）に登録し、「不動産管理会社向けAI自動化コンサルタント」としてプロフィールを作成する。提案文には「初回1社限定で、督促メール自動化システムの構築を半額の40,000円でモニター提供します」と記載する。同時にランサーズ（https://www.lancers.jp）にも同内容で出品する。最初の1社はモニター価格で入ることで実績・口コミ・ケーススタディを得る。これが2社目以降の正規料金獲得への最短ルートだ。

賃貸管理という業種は古く保守的に見えるが、だからこそAIの恩恵をまだ受けていない。担当者が手作業で書いている督促メールを、あなたが設計したシステムが毎朝9時に自動で下書きする——それだけで管理会社の担当者は「魔法を見た」と感じるはずだ。技術的な難易度は高くない。今日、アカウントを3つ作ることから始めよう。
