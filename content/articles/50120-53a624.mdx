# 医療・歯科クリニック向けカルテ・診療報告書作成代行

## なぜ今なのか

日本の医療業界は今、深刻な「事務作業の過負荷」という構造問題を抱えている。厚生労働省の2023年調査によると、医師の週平均労働時間は約60時間を超え、そのうち約30〜40%が電子カルテへの入力・診療録作成・各種報告書の作成に費やされているとされる。要するに、医師は本来の診療行為よりも「書類仕事」に膨大な時間を奪われているのだ。

市場規模の観点でも機会は大きい。日本全国のクリニック（診療所）数は約10万4,000件（2022年時点・厚生労働省）。このうち個人経営や小規模クリニックが8割以上を占め、専任の医療事務スタッフが手薄なケースが多い。歯科クリニックに至っては全国に約6万8,000件存在し、コンビニよりも多いと言われるほど密集している。

なぜ「今」なのか、という問いに対する答えは三つある。

**第一に、2024年の医師の働き方改革の本格施行。** 2024年4月から医師の時間外労働規制（年間960時間上限）が始まった。これにより、従来「医師がやって当たり前」とされていた記録・報告業務をいかに効率化するかが、クリニック経営の喫緊の課題になっている。

**第二に、電子カルテの普及が追い風。** 電子カルテの普及率は一般病院で57.2%（2020年・厚労省）、クリニックでも49.9%に達し、音声入力や音声データをテキスト変換できる環境が整いつつある。デジタルデータを扱いやすくなったことで、AIが介入できる余地が一気に広がった。

**第三に、Claude・GPT-4などの大規模言語モデルの医療文書対応能力の向上。** 医療専門用語・病名・薬品名を正確に扱い、SOAPノート形式（Subjective・Objective・Assessment・Plan）など医療現場の文書フォーマットに沿った生成が実用レベルに達している。

この三つが重なる今こそ、「AIを使った医療文書作成支援」というニッチ高単価マーケットへの参入タイミングだ。

## AIで自動化できる部分の設計

このビジネスの核心は、「音声または簡単なメモ」→「完成度の高い医療文書ドラフト」という変換プロセスをAIで自動化することにある。以下がワークフローの全体像だ。

### 全体フロー概要

```
医師/スタッフが音声録音 or テキストメモを入力
          ↓
音声認識（Whisper API）でテキスト変換
          ↓
Claude API / GPT-4 APIで文書フォーマット化
          ↓
人間レビュワーが確認・修正
          ↓
クライアント（クリニック）に納品
```

### 使用するAIツールと役割

**① OpenAI Whisper API（音声文字起こし）**
診療録作成の最大のボトルネックは「音声→テキスト変換」だ。医師が診察後に口述した内容をWhisper APIに送ると、医療用語を含む高精度な文字起こしが実現できる。日本語対応済みで、料金は$0.006/分と非常に安価。30分の音声でも約18円のコストだ。

**② Claude API（Anthropic）- 文書構造化・フォーマット化**
文字起こされたテキストを、SOAPノート・診療録・紹介状・診断書といった医療文書の形式に整えるのにClaude APIが最適だ。理由は「ハルシネーション（事実でないことを生成する問題）が他のモデルより少ない」という特性にある。医療文書では誤情報は許容されないため、Claude 3 SonnetまたはClaude 3.5 Sonnetを選択する。

システムプロンプトの例：
```
あなたは医療文書作成の専門家です。
入力された診察内容のメモを、以下のSOAPノート形式に整えてください：
- S（Subjective）：患者の主訴・自覚症状
- O（Objective）：医師の観察・検査結果
- A（Assessment）：診断・評価
- P（Plan）：治療方針・処方

医療用語は正確に使用し、曖昧な部分には[要確認]とマークしてください。
```

**③ Make.com（旧Integromat）- ワークフロー自動化**
クライアントが音声ファイルやメモをGoogle DriveやDropboxにアップロードしたら自動的に処理が始まり、完成ドラフトをNotionやGoogle Docsに自動保存するまでの一連を、コードなしで構築できる。月額$9〜のStarter Planで十分対応可能。

## 実際のツール設定（ステップバイステップ）

以下は、Make.com + Whisper API + Claude APIを使ったパイプライン構築手順だ。

**Step 1：各サービスのアカウント・API取得**
1. OpenAI（https://platform.openai.com）にアクセスし、APIキーを発行。Whisper API（モデル名：`whisper-1`）を使用する
2. Anthropic（https://console.anthropic.com）でアカウントを作成し、Claude APIキーを取得。初期クレジット($5)が付与される
3. Make.com（https://www.make.com）に登録し、無料プランで最初のシナリオを作成する

**Step 2：Make.comでGoogle Driveトリガーを設定**
1. Make.comで「新しいシナリオを作成」をクリック
2. 最初のモジュールで「Google Drive」を選択→「Watch Files in a Folder」を選ぶ
3. Googleアカウントを連携し、クライアント専用フォルダ（例：`clinic_A_input`）を指定
4. トリガー間隔：15分ごとに設定

**Step 3：Whisper APIモジュールの追加**
1. Google Driveトリガーの後に「HTTP」モジュールを追加
2. Method：POST、URL：`https://api.openai.com/v1/audio/transcriptions`
3. Headersに`Authorization: Bearer {{OpenAI_APIキー}}`を設定
4. Bodyはmultipart/form-dataで、`file`にGoogle Driveから取得したファイル、`model`に`whisper-1`、`language`に`ja`を指定

**Step 4：Claude APIで文書フォーマット化**
1. HTTPモジュールをさらに追加
2. URL：`https://api.anthropic.com/v1/messages`
3. Headersに`x-api-key`と`anthropic-version: 2023-06-01`を設定
4. Body（JSON）例：
```json
{
  "model": "claude-3-5-sonnet-20241022",
  "max_tokens": 2048,
  "system": "あなたは医療文書作成専門家です。SOAPノート形式で整えてください。",
  "messages": [
    {"role": "user", "content": "{{Step3のWhisper出力テキスト}}"}
  ]
}
```

**Step 5：出力をGoogle Docsに保存**
1. 「Google Docs」モジュールを追加→「Create a Document」
2. ドキュメント名：`{{クリニック名}}_{{日付}}_診療録ドラフト`
3. 保存先フォルダ：レビュワー確認用フォルダを指定
4. シナリオをアクティベートして完成

**Step 6：品質確認フローの構築（人間介在部分）**
完成したドラフトをGoogle Docsで人間が確認する際、コメント機能で修正指示を入れる運用ルールを文書化しておく。Slackに通知を飛ばすなら、Make.comにSlackモジュールを追加するだけで実現できる。

## 自動化できない部分（ここが重要）

AI自動化の精度がいかに上がっても、医療文書という性質上、人間が必ず担わなければならない工程がある。これこそがこのビジネスの「参入障壁」であり、同時に「高単価の正当化根拠」でもある。

**① 医学的正確性の最終確認**
AIは文脈を読んで「それらしい医療用語」を生成するが、特に数値（血圧値・血糖値・投薬量）の誤記・誤変換は致命的だ。「120/80」が「120/8」になっていても文章として成立するため、AIは気づけない。レビュワーは医療知識の基礎を持って数値の妥当性チェックを行う必要がある。

**② 個人情報・患者情報の適切な処理**
患者の氏名・生年月日・病名を含む音声や文書をAPIに送信する際は、個人情報保護法・医療情報の取り扱いガイドライン（厚生労働省「医療情報システムの安全管理に関するガイドライン第6.0版」）に準拠した匿名化・仮名化処理が必要だ。この判断は人間が行う。

**③ クライアント（クリニック）とのコミュニケーション設計**
各クリニックによって電子カルテのシステム（ORCA、M3、Medicom等）が異なり、求める文書フォーマットも異なる。初回の要件ヒアリング・フォーマット設計・フィードバック反映は人間にしかできない関係構築作業だ。

**④ 法的・倫理的判断**
「この記述は診断書として法的に有効か」「守秘義務の観点で問題ないか」という判断は、AIには委ねられない。必要に応じて医療従事者の監修を仰ぐ体制を整えておくことが重要だ。

## 参入のボトルネック

このビジネスには美しい収益モデルがある一方で、正直に言えばいくつかの高い壁も存在する。

**① 医療情報・個人情報の取り扱い問題（最大の壁）**
患者情報は「要配慮個人情報」であり、個人情報保護法の中でも最も厳格な取り扱いが求められる。クリニックとの間で「個人情報取り扱い委託契約」を正式に締結し、外部送信するデータは必ず仮名化・匿名化処理することが必須だ。この契約書を自分で作れるかどうかが最初のハードルになる。

**② 医療知識の習得コスト**
医療文書のレビューに最低限の医学知識は不可欠だ。医療事務経験者・看護師・薬剤師・歯科助手などの資格・経験者は大きく有利。まったくの未経験者は「医療事務通信講座（ユーキャン等）」や「診療報酬請求事務能力認定試験」の学習を3〜6ヶ月かけて行う必要がある。

**③ クリニックへの飛び込み営業の難しさ**
医師・クリニック院長は多忙で、見知らぬサービスの売り込みには極めて冷淡なことが多い。「医師向けコミュニティ」「医療系SNS（m3.com等）」「医療系の展示会・学会」への出展が有効だが、最初の1件を獲得するまでのリードタイムは2〜3ヶ月を見込むべきだ。

**④ AIの出力品質の担保**
プロンプト設計が甘いと、医療用語の誤変換・文書フォーマットのズレが頻発する。実際に運用を始める前に、自分でサンプルデータ（架空の診察メモ）を使って50〜100件分の品質テストを行い、プロンプトを磨く期間が必要だ。

## 最初の一歩

今日から取れる具体的な3ステップを示す。

**Step 1：ツールを無料で触ってみる（今日中にできる）**

- Claude API：https://console.anthropic.com でアカウント作成。初期無料クレジット$5で、架空の診察メモを使ってSOAPノート生成のプロンプトを試す
- Make.com：https://www.make.com/en/register で無料登録（月1,000オペレーションまで無料）。Google Drive連携シナリオを1本作ってみる
- OpenAI（Whisper）：https://platform.openai.com でAPIキー取得。自分の声で「患者が頭痛と発熱を訴えて来院」と録音して文字起こしを試す

**Step 2：医療文書の基礎知識をインプットしながらポートフォリオを作る（1〜2週間）**

- Youtubeで「SOAP note 作り方」「電子カルテ 書き方」で検索し、基本フォーマットを理解する
- 架空のシナリオ（例：「38歳男性、3日前から咳・発熱・倦怠感で来院、体温38.5度、聴診所見清明」）を10件分作成し、Claude APIでSOAPノート化するテストを実施
- 完成した10件のサンプルをGoogle DocsにまとめてポートフォリオURLを作成

**Step 3：最初のクライアントを獲得する（2〜4週間）**

- **クラウドワークス**（https://crowdworks.jp）で「医療事務」「カルテ入力」「診療報告書」のキーワードで案件を検索し、既存案件に応募して実績をまず1件作る
- **ランサーズ**（https://www.lancers.jp）でサービス出品：「AIを活用した診療録ドラフト作成・月額プラン」として月額3万円（10件まで）の試験価格で出品する
- 地域のクリニックへの直接営業：Googleマップで近隣の個人クリニックを5件リストアップし、「医療事務効率化についてのご提案」としてメールDMを送る。件名に「AIを活用した診療録作成時間50%削減の方法」と書くと開封率が上がる

このビジネスは医療×AI×事務代行という三つの希少性が重なる領域だ。最初の1件を獲得できれば、口コミ・紹介による拡大が見込めるリテンション型ビジネスでもある。「医師の時間を守る」という社会的意義も高く、長期的に価格競争に巻き込まれにくい参入障壁の高いニッチだと言えるだろう。
