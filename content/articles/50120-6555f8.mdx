# 建設・リフォーム業向け見積書・提案書作成代行：AI自動化で月収30万円を狙う完全ガイド

---

## なぜ今なのか

建設・リフォーム業界は、日本国内でも有数の「DX後進地帯」だ。国土交通省の調査によると、建設業のIT投資額は製造業の約3分の1にとどまり、現場監督や職人が今なお手書き・Excelで見積書を作成しているケースが珍しくない。

市場規模で言えば、国内リフォーム市場は2023年時点で約7兆円規模（住宅リフォーム推進協議会調べ）。新築需要が人口減少とともに頭打ちになる一方、既存住宅のストック活用としてリフォーム需要は増加傾向にある。しかも2030年には市場が10兆円超になるとも予測されており、業者数・案件数ともに右肩上がりだ。

問題は「担い手不足」にある。建設業就業者の約36%が55歳以上という超高齢化産業であり（国土交通省・建設労働需給調査）、若手が極端に少ない。つまり「見積書を作れる人間がいない」「所長が一人で何十件分の見積書をこなしている」という状態が常態化している。

ここにAIの商機がある。建設見積書の作成は、以下の理由から**AIによる自動化と相性が非常によい**。

- **フォーマットが標準化されている**：工事名・数量・単価・金額・備考という構造はほぼ業界共通
- **単価データが存在する**：国土交通省が公開している「公共工事設計労務単価」や「建設工事市場単価」などの参照データがある
- **繰り返し作業が多い**：キッチン交換・外壁塗装・バスルーム改修など同種の案件が繰り返し発生する
- **提案書の文章量は少ない**：数百〜千数百字程度の提案文であり、AI生成に最適な分量

さらに、Upworkでは"Construction Estimate Writer"や"Proposal Writer for Contractors"というカテゴリで案件が恒常的に出回っており、英語圏では既に成立したビジネスモデルとなっている。日本では同等のサービスがクラウドワークスやランサーズにほぼ存在しない。**今がブルーオーシャンに入る最後のタイミング**といえる。

## AIで自動化できる部分の設計

全体のワークフローは以下のように設計する。

```
【入力】クライアントからのヒアリングシート（Google Forms）
　　↓
【処理1】Make.comが回答データを受け取り、Claude APIへ送信
　　↓
【処理2】Claude APIが見積書の下書きテキストと提案文を生成
　　↓
【処理3】Make.comがGoogle SheetsまたはNotionに出力
　　↓
【処理4】あなたが数値確認・微調整
　　↓
【出力】PDFに変換してクライアントへ納品
```

**使用するツール一覧**

| ツール | 用途 | 月額コスト |
|---|---|---|
| Google Forms | ヒアリング入力 | 無料 |
| Make.com（旧Integromat） | 自動化ハブ | 無料〜$9 |
| Claude API（Anthropic） | 文章・構成生成 | 従量課金（1件あたり約$0.1〜0.3） |
| Google Sheets | 見積データ管理 | 無料 |
| Adobe Acrobat / DocuSign | PDF化・電子署名 | $14.99〜/月 |
| Notion | 案件管理ダッシュボード | 無料〜$10 |

**Claude APIを選ぶ理由**：GPT-4と比較して長文の構造化出力（表形式・箇条書き混在）の精度が高く、「見積書」という特定フォーマットへの整形が得意。また、日本語の文体制御（丁寧語・業界用語）もGPT-4と同等以上の品質を発揮する。

**ヒアリングシートで収集すべき情報**

Google Formsに以下の項目を設定しておく。

1. 工事種別（外壁塗装 / キッチン交換 / 浴室改修 / 屋根工事 / 内装工事 など）
2. 施工面積または施工箇所数
3. 使用希望材料・グレード（標準 / 中級 / 高級）
4. 施工場所の築年数・現状の問題点
5. 希望完工時期
6. 見積書に含めたい特記事項（保証期間・支払条件など）
7. 提案書のトーン（価格重視 / 品質重視 / スピード重視）

これらの情報をClaude APIに渡すことで、**人間が一から考えなくても業界水準の見積書・提案書が自動生成**される。

## 実際のツール設定（ステップバイステップ）

### STEP 1：Google Formsでヒアリングシートを作成する

1. Googleアカウントにログインし、Google Forms（forms.google.com）で新規フォームを作成
2. 上記7項目を質問として追加。「工事種別」はプルダウン式、面積は数値入力、その他は段落テキストで設定
3. フォームの回答をGoogle Sheetsに連携：「回答」タブ→「スプレッドシートにリンク」をクリック
4. スプレッドシートのシート名を「raw_input」に変更しておく

### STEP 2：Make.comのシナリオを構築する

1. Make.com（make.com）に無料登録
2. 「新規シナリオ作成」→トリガーとして「Google Sheets - Watch New Rows」を選択
3. 対象スプレッドシートとシート「raw_input」を指定。ポーリング間隔は15分に設定
4. 次のモジュールとして「HTTP - Make a request」を追加
   - URL：`https://api.anthropic.com/v1/messages`
   - メソッド：POST
   - ヘッダー：`x-api-key: YOUR_CLAUDE_API_KEY`、`anthropic-version: 2023-06-01`、`Content-Type: application/json`
5. Body（JSON）に以下のような構造を設定する：

```json
{
  "model": "claude-opus-4-5",
  "max_tokens": 2048,
  "messages": [
    {
      "role": "user",
      "content": "あなたは建設・リフォーム業の見積書作成の専門家です。以下のヒアリング情報をもとに、①工事項目別の見積表（工事名・仕様・数量・単価・金額・備考の6列）と②A4一枚程度の提案書本文（会社への信頼感を醸成する文章）を作成してください。\n\n工事種別：{{工事種別}}\n施工面積：{{施工面積}}\n材料グレード：{{材料グレード}}\n築年数・現状：{{現状}}\n特記事項：{{特記事項}}\n提案書トーン：{{トーン}}"
    }
  ]
}
```

6. Claude APIの返答（content[0].text）をパース：「JSON - Parse JSON」モジュールを追加
7. 出力先として「Google Sheets - Add a Row」モジュールを追加し、生成された見積表テキストと提案文をそれぞれの列に書き出す
8. 最後に「Gmail - Send an Email」モジュールで自分宛に「新規案件が処理されました」という通知メールを送信

### STEP 3：Google Sheetsでの確認・修正テンプレートを整備する

1. 出力用スプレッドシートに「draft_output」シートを作成
2. A列：工事名、B列：仕様、C列：数量、D列：単価、E列：金額（=C*D の数式を設定）、F列：備考
3. 合計金額・消費税・値引き後合計の計算式をあらかじめ最下行に設定
4. スプレッドシートの「ファイル→ダウンロード→PDF」機能でそのままPDF化できるようにレイアウトを調整

### STEP 4：Claude APIのプロンプトをチューニングする

初回は必ず5〜10件分のサンプルで出力テストを行う。特に単価の精度を確認し、プロンプトに「国土交通省の公共工事設計労務単価（2024年度版）を参考に、東京都内の市場相場で算出すること」という指示を加えると精度が大幅に向上する。また、Few-shot学習として実際の見積書フォーマット例をプロンプトに2〜3件埋め込むと、出力フォーマットの安定性が増す。

## 自動化できない部分（ここが重要）

AIと自動化が担えるのは「構造化された情報の整形と文章生成」までだ。以下の部分は**必ず人間が介在**しなければならず、ここが競合との差別化ポイントとなる。

**①現場特有の条件反映**
AIはヒアリングシートの情報しか見ていない。実際には「既存タイルの撤去が想定より難しい」「搬入経路が狭く追加費用が発生する」といった現場固有の事情がある。建設業界の経験者であれば「この条件ならこの工事は割増になる」という判断ができるが、AI単体では不可能だ。クライアントとの電話ヒアリングで補足情報を引き出すスキルが求められる。

**②単価の地域・時期調整**
国土交通省の単価データはあくまで参考値であり、資材費の高騰（2023〜2024年は建材価格が約15〜30%上昇）や地域差を反映していない。最終的な単価調整は人間がチェックしなければ「安すぎて受注しても赤字」という事態を招く。

**③クライアントとの関係構築・折衝**
見積書を送った後の価格交渉、工事内容の変更相談、クレーム対応はすべて人間の領域だ。建設業のクライアントは「人との信頼関係」を非常に重視する傾向があり、チャットボットや自動返信では対応できない。

**④法的チェック・資格要件の確認**
特定の工事（電気工事・ガス工事・特殊建築など）は資格保有者のみが施工可能であり、見積書にその旨を正確に記載しなければならない。AIは資格要件を誤って省略・記載することがあるため、最終確認は必須だ。

**⑤ブランディングと差別化文章**
「なぜ他社ではなくこの会社に頼むべきか」という説得力のある提案文は、そのクライアントの強みや実績を深く理解していないと書けない。AIが生成するのは汎用的な文章であり、クライアント固有のアピールポイントを反映させるには人間のインタビューと編集が必要になる。

## 収益モデルの試算

**相場の根拠**

- **Upwork**："Construction Proposal Writer"の相場は$75〜$200/件。実績のあるフリーランサーは$150〜$250/件で受注している（2024年現在の検索結果より）
- **Fiverr**："Construction Estimate"カテゴリでは$50〜$150/件が中心価格帯。Level 2以上のセラーは$100〜$300/件のパッケージを設定している
- **クラウドワークス・ランサーズ**：類似サービスは現状ほぼ存在しないため比較困難だが、「提案書・企画書作成」カテゴリの相場は5,000〜30,000円/件。建設業特化・AIツール活用を明示すれば15,000〜50,000円/件の設定は十分に正当化できる

**料金プランの設計**

| プラン | 内容 | 単価 |
|---|---|---|
| ライトプラン | 見積書のみ（5項目以内） | 8,000円 |
| スタンダードプラン | 見積書＋提案書A4一枚 | 18,000円 |
| プレミアムプラン | 見積書＋提案書＋競合比較資料＋修正2回 | 35,000円 |
| 月額顧問プラン | 月10件まで見積書＋提案書作成 | 80,000円/月 |

**月収試算**

最も現実的な収益モデルは「月額顧問プラン」の積み上げ型だ。

```
月額顧問プラン：80,000円 × 3社 ＝ 240,000円
スポット案件（スタンダード）：18,000円 × 5件 ＝ 90,000円
─────────────────────────
月収合計：330,000円
```

**作業時間の見積もり**

- ヒアリングシート確認・補足連絡：1件あたり30分
- AI生成内容のチェック・修正：1件あたり20〜40分
- PDF化・納品：1件あたり10分
- 顧問3社（月30件）＋スポット5件＝月35件

合計作業時間：35件 × 約1時間 ＝ **月35時間**（週約9時間）

副業として十分に現実的な時間内で月収33万円を達成できる計算になる。

## 参入のボトルネック

正直に書く。この副業には以下の障壁が存在する。

**①建設業の知識が必要**
工事種別・建材の種類・施工工程・業界用語（積算・拾い出し・歩掛かりなど）の基礎知識がないと、AIの出力を正しくチェックできない。知識がまったくゼロの場合、最低でも2〜3ヶ月の自己学習期間が必要だ。ただし、建設関連の職場経験者・設計士・インテリアコーディネーターなら即戦力になれる。

**②最初のクライアント獲得が最大の壁**
建設業者はいまだに「知人の紹介」「既存業者への依頼」を重視する傾向がある。プラットフォーム経由での新規獲得は時間がかかる。最初の1〜3社は地域の工務店・リフォーム会社への直接営業（飛び込みまたはDM）が最も効果的だが、精神的なハードルは高い。

**③Claude APIのコストと精度管理**
月35件の処理でAPIコストは月$10〜15程度と低コストだが、プロンプトの設計が悪いと単価を大きく外した見積書が生成される。最初の1ヶ月は必ず全件を建設業者にフィードバックしてもらいながら精度改善に集中する必要がある。

**④法的なグレーゾーン（軽微な注意点）**
建設業法では「建設工事の請負契約」に関して一定の制限があるが、あくまで「見積書・提案書の作成代行」であれば該当しない。ただし、見積書の内容について「この金額で受注できる」という保証を与えるような文言は避けるべきだ。また、クライアントの社名・工事内容・金額情報は機密情報にあたるため、NDA（秘密保持契約）を必ず締結することを推奨する。

**⑤競合の参入リスク**
現状はブルーオーシャンだが、この手法が広まれば類似サービスが増える。差別化のためには「建設業専門のテンプレートライブラリ（100種類以上）」「施工写真つき提案書」「動画見積もり説明資料」など、付加価値を常にアップデートし続ける必要がある。

## 最初の一歩

今日から72時間以内にできる具体的な3ステップを示す。

**STEP 1：ツールを揃える（今日中・費用ほぼゼロ）**

1. **Claude API**：Anthropicの公式サイト（console.anthropic.com）にアクセスし、APIキーを取得。$5分のクレジットが付与されるので無料で始められる
2. **Make.com**：make.com に無料登録。月1,000オペレーション・2シナリオまで無料で使える
3. **Google Forms & Sheets**：Googleアカウントがあれば即日利用可能（forms.google.com）
4. **Notion**：案件管理用。notion.so に無料登録し、「建設見積案件管理」テンプレートを作成

**STEP 2：サービスページを公開する（2〜3日以内）**

1. **クラウドワークス**（crowdworks.jp）に登録し、「建設・リフォーム業向け見積書・提案書作成代行（AI活用で最速翌日納品）」というタイトルでサービスを出品
2. **ランサーズ**（lancers.jp）にも同様のサービスを登録。掲載は無料
3. プロフィール欄に「建設業のDX支援・見積書作成代行の専門家」と明記し、実績サンプルとして架空案件（「外壁塗装120㎡の見積書サンプル」）をPDFで添付する
4. 最初の3件は**50%オフのモニター価格（例：スタンダードプランを9,000円）**で受け付け、レビュー獲得を最優先にする

**STEP 3：最初のクライアントに直接アプローチする（今週中）**

1. Googleマップで「リフォーム 〇〇市」と検索し、レビュー数が少ない（＝デジタル対応が弱い）工務店・リフォーム会社を10〜20社リストアップ
2. 各社のホームページから問い合わせフォームまたはメールアドレスを取得
3. 以下のような営業メールを送信する：

> 「はじめまして。見積書・提案書作成の代行サービスを提供しております〇〇と申します。AIを活用することで、通常3〜5日かかる見積書を最短翌日でご納品できます。初回のみ無料でお試しいただける件数を設けておりますので、ご興味があればお気軽にご連絡ください。」

この3ステップを実行するのに必要なコストは**ほぼゼロ**。必要なのは時間と行動力だけだ。建設業界のDX化は今まさに始まったばかりであり、最初に市場に根を張った人間が圧倒的に有利なポジションを取れる。「今日動かない理由」を探すより、「今日動く最小の一歩」を踏み出すことが最も重要だ。
