# 士業（行政書士・社労士）向け書類作成代行｜AIで「書類の工場」を作る方法

---

## なぜ今なのか

日本の士業市場は、構造的な「需給ミスマッチ」が深刻化している。

行政書士の登録者数は現在約53,000人（日本行政書士会連合会、2024年）だが、そのうち実際に事務所を経営して生計を立てているのは半数以下とも言われる。社労士（社会保険労務士）は約43,000人が登録しているが、こちらも同様に稼働率の問題を抱えている。

問題は「人数が少ない」ことではなく、「単純反復業務に時間を食われすぎている」ことだ。

行政書士が取り扱う業務のうち、建設業許可申請、在留資格（ビザ）申請、会社設立書類などは、基本的な書式が決まっており、ヒアリング内容を所定の書式に落とし込む作業が大半を占める。社労士においても、就業規則の雛形作成、36協定届、社会保険の各種手続き書類など、「情報を集めてフォーマットに当てはめる」作業は全体業務の60〜70%に達するという調査結果もある（日本社会保険労務士会連合会の業務実態調査より）。

ここにAI自動化の商機がある。

具体的な数字を見てみよう。行政書士事務所の平均的な月間書類処理件数は30〜100件。1件あたりの書類作成に2〜5時間かかるとすると、月間60〜500時間が「書類を書く作業」に消えている計算になる。時給換算すれば、この時間を削減するだけで事務所の収益性は劇的に改善する。

AI技術の進歩も追い風だ。GPT-4やClaudeは2023年以降、日本語の文書生成精度が飛躍的に向上し、法令用語を含む行政文書の草稿生成も現実的なレベルに達している。さらに、n8nやMake.comといったノーコード自動化ツールの普及により、プログラミング経験がほぼなくてもAPIを組み合わせたワークフローを構築できるようになった。

副業として参入する場合、あなたは「AIを使って書類作業を高速化するアシスタント」として士業事務所と契約する形になる。法的資格は不要（後述）。月10〜20時間の稼働で5〜15万円の副収入を狙えるポジションだ。

## AIで自動化できる部分の設計

このビジネスの核心は「ヒアリング情報→書類草稿」のパイプラインをAIで構築することだ。全体ワークフローを設計する前に、どの作業がAI化できるかを整理しておこう。

**自動化対象となる主要書類の例：**
- 建設業許可申請書（国土交通省様式）
- 在留資格変更許可申請書
- 会社設立時の定款草稿
- 就業規則（基本部分のドラフト）
- 雇用契約書テンプレートのカスタマイズ
- 36協定届の計算・記入支援
- 補助金申請書（事業計画部分）

**推奨ワークフロー全体像：**

```
顧客ヒアリングフォーム（Google Forms / Typeform）
　　↓
Make.com / n8n でトリガー受信
　　↓
Claude API / GPT-4 API で書類草稿生成
　　↓
Google Docs / Notion に出力
　　↓
人間（あなた or 士業者）がレビュー・修正
　　↓
PDF化して納品
```

**各ツールの役割分担：**

**Typeform**（ヒアリング収集）：条件分岐ロジックが使えるため、「会社設立の場合はA経路、建設業許可の場合はB経路」という形でヒアリング項目を動的に変えられる。無料プランでも基本機能は使用可能。

**Make.com**（自動化ハブ）：Typeformの回答を受け取り、Claude APIに送り、結果をGoogle Docsに書き出すまでの一連の流れを「シナリオ」として管理する。月額$9（コアプラン）から使用可能。

**Claude API**（文書生成エンジン）：Anthropic社のClaude 3.5 SonnetはAPIで利用でき、長文の構造化文書生成に特に強い。トークンあたりの価格はClaude 3.5 Sonnetで入力$3/百万トークン、出力$15/百万トークン。1件の書類生成コストは通常$0.10〜$0.50程度に収まる。

**Google Docs API**（出力先）：生成された草稿を特定のテンプレートに流し込む形で使用。Make.comとの連携モジュールが標準提供されている。

## 実際のツール設定（ステップバイステップ）

ここでは「就業規則ドラフト作成」を例に、実際のセットアップ手順を説明する。

### STEP 1：Claude APIキーの取得

1. `https://console.anthropic.com` にアクセスし、アカウントを作成
2. 「API Keys」メニューから新しいキーを発行（名前は「gyosei-shoshi-system」など分かりやすく）
3. 初回は$5のクレジットが付与される。本格稼働前にこれでテスト可能
4. キーは後でMake.comに入力するので安全な場所に保管

### STEP 2：Typeformでヒアリングフォームを作成

1. `https://www.typeform.com` でアカウント作成（無料プランでOK）
2. 「New typeform」→「Start from scratch」
3. 以下の質問を追加する：
   - 会社名・代表者名（Short Text）
   - 業種・事業内容（Long Text）
   - 従業員数（Number）
   - 労働時間制度の種類（Multiple Choice：通常勤務/フレックス/裁量労働）
   - 休日・休暇の希望（Checkboxes）
   - 試用期間の有無（Yes/No）
   - 特記事項（Long Text）
4. 「Connect」タブからWebhookを設定。URLは後でMake.comから取得する

### STEP 3：Make.comでシナリオを構築

1. `https://www.make.com` でアカウント作成
2. 「Create a new scenario」をクリック
3. **最初のモジュール**：「Webhooks」→「Custom webhook」を選択し、生成されたURLをTypeformのWebhook設定に貼り付け
4. **2番目のモジュール**：「HTTP」→「Make an API Key Auth Request」を選択
   - URL: `https://api.anthropic.com/v1/messages`
   - Method: POST
   - Headers:
     - `x-api-key`: [取得したClaudeのAPIキー]
     - `anthropic-version`: `2023-06-01`
     - `content-type`: `application/json`
   - Body（RAW JSON）:
     ```json
     {
       "model": "claude-3-5-sonnet-20241022",
       "max_tokens": 4000,
       "messages": [
         {
           "role": "user",
           "content": "以下の情報をもとに、労働基準法に準拠した就業規則のドラフトを作成してください。\n会社名: {{1.company_name}}\n業種: {{1.industry}}\n従業員数: {{1.employee_count}}\n労働時間制度: {{1.work_hours}}\n休日・休暇: {{1.holidays}}\n試用期間: {{1.probation}}\n特記事項: {{1.notes}}\n\n出力は実際に使用できる就業規則の文章形式で、第1条から順番に記載してください。"
         }
       ]
     }
     ```
5. **3番目のモジュール**：「Google Docs」→「Create a Document from a Template」
   - 事前にGoogle Docsで就業規則のテンプレート（会社名・日付などをプレースホルダーにしたもの）を作成
   - 該当のテンプレートIDを指定し、Claudeの出力（`{{2.content[0].text}}`）を本文エリアに流し込む
6. **4番目のモジュール**：「Gmail」→「Send an Email」
   - 担当士業者のアドレスに「レビュー依頼：[会社名]様就業規則ドラフト完成」の件名でGoogle DocsのリンクをURL付きで送信

### STEP 4：プロンプトの精度チューニング

Claude APIへのプロンプトは最初から完璧にはならない。最低10件の実案件で出力を確認し、以下の点を改善していく：

- **法令条文の参照指示を追加**：「労働基準法第89条の必要記載事項を漏れなく含めること」などを明示
- **Few-shotサンプルの挿入**：良質な就業規則の例（個人情報除去済み）をプロンプトの冒頭に貼り付けると精度が大幅に向上する
- **チェックリスト出力の要求**：本文と合わせて「記載漏れチェックリスト」も出力させると士業者のレビューが楽になる

## 自動化できない部分（ここが重要）

AIワークフローがどれだけ精巧でも、以下の部分は必ず人間が担う必要がある。この「自動化できない部分」こそが、あなたのサービスの差別化ポイントであり、単なる「AIツール使い」との差を生む。

**1. 最終的な法的適合性の確認**
AIが生成した書類は、常に最新の法改正に対応しているとは限らない。特に労働関係法令は毎年改正があるため、書類の最終確認は必ず士業資格者が行う必要がある。あなたの役割は「草稿を高速生成すること」であり、法的責任は資格者が負う構造を契約書で明確にすること。

**2. 個別事情の深掘りヒアリング**
フォームで収集できる情報には限界がある。「フランチャイズ経営で複数の就業形態が混在する」「外国人労働者を雇用予定で二か国語対応が必要」といった複雑な状況は、電話・Zoom等でのヒアリングが不可欠。この部分はむしろあなたの付加価値になる。

**3. 申請書類の官公庁への提出プロセス管理**
行政書士業務の場合、書類を作るだけでなく申請の進捗管理・補正対応・窓口との折衝が発生する。この部分は完全に士業者の領域だが、あなたが補正依頼時の書類修正を素早く対応できる体制を作れると重宝される。

**4. クライアントとの信頼関係構築**
士業事務所の顧問契約は「この先生（事務所）なら安心」という信頼が土台。あなたはバックオフィスの効率化パートナーとして、士業者の信頼を支える立場に徹すること。

**5. イレギュラー対応**
建設業許可で「経営業務の管理責任者の経験年数が微妙」「在留資格で過去に不法就労の疑義がある」といったグレーゾーン案件は、AIの出力をそのまま使うと重大なリスクになりえる。フラグを立てて人間にエスカレーションするルール設計が必要。

## 参入のボトルネック

正直に障壁を書く。これを読んでから始めるかどうかを判断してほしい。

**1. 初回クライアント獲得の難しさ**
最大の壁はここだ。士業事務所は保守的な業界であり、「よく分からないAIツールを使ったサービス」には警戒心が強い。特に顧問先企業の機密情報を扱う業態のため、「信頼できる人間かどうか」が最重要視される。最初の1〜2件は、知人の士業者や士業向けコミュニティからの紹介でないと獲得は難しい。

**2. ドメイン知識の習得コスト**
行政書士が扱う書類の種類は数百種類以上あり、各書類の記載ルール・添付書類・提出先を一定程度理解していないと、適切なプロンプト設計ができない。最低でも主要10種類程度の書類については、実物を入手して構造を理解しておく必要がある（官公庁のウェブサイトで多くは入手可能）。

**3. 情報セキュリティへの対応**
士業事務所が取り扱う情報（外国人の在留資格情報、法人の財務情報、雇用契約内容など）は個人情報保護法・マイナンバー法の対象になる場合が多い。Claude APIを使う場合、デフォルトではデータが学習に使われる可能性があるため、Anthropicのエンタープライズ向けプライバシー設定（APIキー利用時はデータを学習に使用しない旨のポリシー確認）を事務所に説明できるようにしておく必要がある。また、クライアントとの間でNDA（秘密保持契約）を必ず締結すること。

**4. 法的な業務範囲の理解**
これは重要な注意点だ。行政書士法・社会保険労務士法は、有資格者でなければ「業として」特定の書類を「作成」することを禁じている。副業者であるあなたが行うのは、あくまで「草稿の生成補助・データ入力補助」であり、最終的な書類の作成責任は士業者が担う構造を明確にしなければならない。この点を曖昧にしたビジネスモデルは法的リスクがある。「AIツールオペレーター」「業務効率化コンサルタント」としての立場を明確にすること。

**5. プロンプトの継続的メンテナンス**
法改正があるたびにプロンプトを更新する必要がある。これを怠ると、古い法令に基づいた書類草稿が生成され続けるリスクがある。
