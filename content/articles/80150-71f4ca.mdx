# 弁護士・法律事務所向け契約書レビュー・作成代行｜AI自動化で月30万円を狙う完全設計書

---

## なぜ今なのか

日本の法律市場は2023年時点で約3.5兆円規模（日本弁護士連合会・法務省統計より推計）。しかし弁護士の絶対数は約4.7万人（2024年時点）で、中小企業や個人事業主が気軽に法的サービスにアクセスできる状況には程遠い。

問題の核心は「コストと時間」だ。一般的な弁護士事務所に契約書レビューを依頼すると、1件あたり3万〜15万円・納期は3〜10営業日が相場。中小企業の経営者にとってこれは非現実的に高い。結果として「ノーチェックでサインする」「テンプレをそのまま使う」という危険な実態が広がっている。

ここにAIの商機がある。

GPT-4やClaudeの登場により、契約書の**条文解析・リスク抽出・修正案の提示**が劇的に低コスト・短時間で実現できるようになった。特に2023〜2024年にかけてClaude 3 OpusおよびGPT-4oが長文コンテキスト（200K〜128Kトークン）に対応したことで、50ページ超の英文契約書でも一括処理が可能になっている。

需要サイドを見ると、クラウドワークスやランサーズでは「契約書作成」「NDA作成」「利用規約作成」といったカテゴリの案件が月間300〜500件以上投稿されており、常に供給不足の状態。Upworkでも"Contract Review"カテゴリのフリーランサー単価は**$50〜$200/件**（日本語対応・英日バイリンガル案件は$80〜$300/件）で推移している。

さらに重要なのが「弁護士事務所そのものがAIを活用したい」という需要だ。法律事務所はリサーチアシスタントや事務スタッフの採用に苦労しており、**ドラフト作成の下請け・AI活用のオペレーション設計**を外部委託したいニーズが急増している。これが2024年現在の最大の参入機会である。

## AIで自動化できる部分の設計

全体ワークフローは「受注→文書取り込み→AI解析→レポート生成→納品」の5フェーズで構成する。各フェーズで使うツールを以下に整理する。

### フェーズ1：受注・ファイル受取（自動化率：95%）

- **Typeform** または **Tally.so** でクライアントの依頼フォームを作成
- フォームには「契約書の種類（NDA/業務委託/売買など）」「依頼内容（レビュー/作成/修正）」「懸念点の自由記述」「ファイルアップロード」欄を設置
- 送信トリガーで **Make.com** のシナリオが起動し、ファイルをGoogle Driveの専用フォルダに自動格納

### フェーズ2：文書解析エンジン（自動化率：85%）

- **Claude API（claude-3-5-sonnet-20241022）** をコアエンジンとして採用
- PDFファイルはまず **Adobe PDF Extract API** または **pdfplumber（Python）** でテキスト抽出
- 抽出テキストをClaude APIに投げる際、**システムプロンプト**に以下を埋め込む：
  - 「あなたは日本法に精通した法務レビューアシスタントです。以下の契約書を読み、①リスク条項の抽出、②有利・不利な条件の整理、③修正推奨箇所の列挙、④修正文案の提示、⑤総評コメントをJSON形式で返してください」

### フェーズ3：レポート自動生成（自動化率：90%）

- Claude APIのJSON出力を **Make.com** が受け取り、**Google Docs API** 経由で定型レポートテンプレに自動流し込み
- レポート構成：表紙（依頼者名・日付・契約書タイトル）→エグゼクティブサマリー→リスクマトリクス（高/中/低）→条項別詳細コメント→修正案→免責事項
- 生成されたGoogleドキュメントをPDF化し、Googleドライブの共有リンクをSlackまたはメールで自動送信

### フェーズ4：品質チェック（人間が介入：必須）

ここは後述の「自動化できない部分」で詳述するが、最終確認は人間が行う。所要時間は契約書1件あたり**15〜40分**に圧縮できる。

### フェーズ5：納品・請求（自動化率：95%）

- **Stripe** でオンライン決済を自動処理
- 納品メールは **SendGrid API** でMake.comから自動送信
- 案件ステータス管理は **Notion Database** にMake.comが自動記録

## 実際のツール設定（ステップバイステップ）

### Step 1：Claude APIキーの取得と設定

1. [https://www.anthropic.com/](https://www.anthropic.com/) にアクセスし、Anthropicアカウントを作成
2. 「API Keys」メニューから新規キーを発行（課金設定：claude-3-5-sonnetは入力$3/MTok・出力$15/MTok）
3. APIキーをメモ帳には保存せず、後述するMake.comの「Connections」に直接登録

### Step 2：Make.comのシナリオ構築

1. [https://www.make.com/](https://www.make.com/) にアクセスし、無料アカウント作成（月1,000オペレーション無料）
2. 「Create a new scenario」をクリック
3. **トリガーモジュール**：「Typeform → Watch Responses」を選択し、先ほど作ったフォームと接続
4. **モジュール2**：「Google Drive → Upload a File」でアップロードされたPDFを指定フォルダに保存
5. **モジュール3**：「HTTP → Make a request」でClaude APIを呼び出す
   - URL：`https://api.anthropic.com/v1/messages`
   - Method：POST
   - Headers：`x-api-key: YOUR_API_KEY`、`anthropic-version: 2023-06-01`、`content-type: application/json`
   - Body（JSON）：modelに`claude-3-5-sonnet-20241022`、max_tokensに`4096`、messagesにユーザーの契約書テキストを格納
6. **モジュール4**：「Google Docs → Create a Document from Template」でレポートテンプレに値を流し込み
7. **モジュール5**：「Gmail → Send an Email」で納品リンクをクライアントに自動送信
8. 「Run once」でテスト実行し、各モジュールの出力を確認

### Step 3：プロンプトの最適化

Make.comのHTTPモジュールのBodyに埋め込むシステムプロンプトを以下のように設定する。

```
System: あなたは日本法（民法・商法・下請法・労働法）に精通した
法務レビューアシスタントです。

User: 以下の契約書をレビューし、必ず下記JSON形式で回答してください。
{
  "summary": "契約書の概要（200字以内）",
  "risk_high": ["高リスク条項のリスト"],
  "risk_medium": ["中リスク条項のリスト"],
  "risk_low": ["低リスク条項のリスト"],
  "modifications": [
    {"clause": "対象条項名", "current": "現行文", "suggested": "修正案", "reason": "修正理由"}
  ],
  "overall_comment": "総評（300字以内）"
}

契約書本文：{{extracted_text}}
```

### Step 4：n8nをセルフホストで使う場合（コスト削減版）

4. 月額コストはRailwayの従量課金で**$5〜$15/月**に抑えられる

## 自動化できない部分（ここが重要）

ここがビジネスの本質的な差別化ポイントであり、「AIに仕事を奪われる」ではなく「AIを使いこなす人間が価値を持つ」理由でもある。

**① 法的判断の最終責任**
AIは「この条項は甲に不利」と指摘できるが、「この案件でクライアントはその不利を受け入れるべきか否か」というビジネス判断と法的アドバイスは人間が行う。特に**弁護士法72条（非弁行為の禁止）**の観点から、具体的な法的アドバイスは弁護士資格者が行わなければならない。このモデルで参入する場合は「法的アドバイスではなくリスクの可視化サービス」と明確に位置づけるか、弁護士と業務提携することが必須。

**② 業種固有の商慣習の把握**
建設業の下請け契約、IT業界のSaaS利用規約、音楽・映像業界のライセンス契約は、それぞれに業界特有の慣行がある。AIはその慣行を文脈として読めないケースがあり、「法的には問題ないが業界では通用しない条項」の判断は人間の経験値が必要。

**③ クライアントのビジネス文脈の読み取り**
「相手企業との力関係」「この契約が長期取引の入り口かどうか」「担当者の意図」などはフォームや契約書文面からは読み取れない。初回ヒアリング（30分のビデオ通話）は必ず人間が行い、コンテキストをAIに渡す前に整理する作業が必要。

**④ 最終レポートの文章品質チェック**
Claude APIの出力は高品質だが、敬語の使い方・クライアントへの配慮表現・誤訳（英日バイリンガル案件）などは必ず人間がレビューする。この工程を省くと信頼を損なう。

**⑤ 新しい法改正への即時対応**
2024年の下請法改正や2025年施行予定の法令変更など、最新の法改正をプロンプトに反映する作業は人間が定期的に行う必要がある。

## 参入のボトルネック

正直に言う。この領域にはいくつかの現実的な障壁がある。

**① 弁護士法72条（非弁行為）のリスク**
最大の障壁。「法律事務」を報酬をもらって行えるのは弁護士・司法書士・行政書士などに限られる。「AIを使ったリスク可視化・情報提供サービス」と「法的アドバイス」の境界線は実務上グレーゾーンも多い。**安全な参入方法は2つ：弁護士・行政書士との業務提携、もしくは「情報提供・翻訳・書式整備サービス」として明確に位置づけること。**

**② 初期の信頼構築**
法律文書は「信頼が命」のジャンル。Upwork・クラウドワークス等での実績ゼロから始める場合、最初の5〜10件は相場の半額以下でポートフォリオを作る必要がある。この期間（1〜2ヶ月）を乗り越えられるかが最初の関門。

**③ 専門知識のベースライン**
AIが優れた出力を出すためには、人間がアウトプットの質を評価できる必要がある。最低限、民法（契約編）・下請法の基礎知識は独学で習得すること。**「ビジネス実務法務検定3級」（CBT方式・受験料7,480円）の取得が最短の信頼担保になる。**

**④ プロンプトエンジニアリングの習得コスト**
精度の高い法務プロンプトを構築するには20〜30時間の試行錯誤が必要。最初から完璧なアウトプットは出ない。

**⑤ 英語案件への対応**
Upworkでの高単価案件は英語が大前提。英日バイリンガルの法律用語に慣れるまで、英語案件はハードルが高い。最初は国内（クラウドワークス・ランサーズ）から始めるのが現実的。

## 最初の一歩

今日から72時間以内にできる具体的な3ステップを示す。

### ステップ1：AIの精度を自分で体験する（今日中・コスト0円）

1. **Claude.ai**（https://claude.ai/）に無料アカウントを作成
2. インターネット上で公開されている「NDAサンプル」（検索：「秘密保持契約書 サンプル PDF」）をテキストでコピー
3. Claudeに以下のプロンプトを投げる：「この契約書を法務担当者の視点でレビューし、リスク条項・修正推奨箇所・修正案を日本語で詳しく教えてください。[契約書テキストをペースト]」
4. 出力された結果を見て、自分が人間として何を付加できるか考える。これがあなたのビジネス設計の起点になる。

### ステップ2：サービスを登録する（48時間以内・コスト0円）

1. **クラウドワークス**（https://crowdworks.jp/）に「サービス出品者」として登録
2. プロフィールに「AI活用による契約書レビュー・作成サービス。ビジネス実務法務検定取得（または勉強中）。24時間以内に初稿を納品」と記載（資格がなくても「勉強中」と正直に書く方が誠実で信頼される）
3. 最初のサービス出品価格：NDAレビューを**5,000円（通常価格の1/3）**で設定し、「初期モニター3名限定」とする
4. 同様に**ランサーズ**（https://www.lancers.jp/）にも同内容で登録（2プラットフォーム同時出品で露出を最大化）

### ステップ3：自動化パイプラインの骨格を作る（72時間以内・コスト$0〜$16/月）

1. **Make.com**（https://www.make.com/）に無料アカウントを作成
2. **Tally.so**（https://tally.so/）で依頼受付フォームを無料作成（契約書の種類・懸念点・ファイルアップロード欄を設置）
3. Make.comで「Tally → Google Drive保存 → GmailでAI解析結果を自分に送信」という最小構成のシナリオを組む（Claude APIキーの取得はAnthropicサイトで無料登録後すぐ可能）
4. この骨格が完成すれば、最初の案件を受注した瞬間から自動化が機能する

**補足：弁護士との提携を早期に検討する**

ビジネスが軌道に乗ったタイミングで、**弁護士ドットコム**（https://www.bengo4.com/）や地域の弁護士会の紹介制度を通じて提携弁護士を探すことを強く推奨する。「AI処理は私が担当・最終法的確認は弁護士が担当」という分業モデルにすることで、非弁リスクを排除しながら単価を2〜3倍に引き上げることが可能になる。
