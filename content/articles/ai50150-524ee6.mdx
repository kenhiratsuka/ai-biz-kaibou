# 採用試験・資格試験対策講座の問題作成・解説文生成代行｜AIビジネス解剖室

---

## なぜ今なのか

日本の資格試験市場は年間約3,000億円規模（矢野経済研究所調べ）とされており、宅建士・FP・社会保険労務士・医療系国家試験・公務員試験・TOEIC対策など、毎年数百万人が何らかの試験対策に費用を投じている。加えて、採用試験のSPI対策・玉手箱対策・CAB/GAB対策など企業の新卒採用市場も巨大だ。少子化にもかかわらず受験者数が横ばいを維持しているのは、転職市場の拡大と「学び直し」需要の増大が背景にある。

この市場で深刻な問題として浮上しているのが「問題制作者不足」だ。良質な模擬問題を1問作るには、出題範囲の精通・正確な法令知識・誤答選択肢の巧妙な設計・明快な解説文の執筆、という高度なスキルセットが必要になる。従来は予備校の専任講師や出版社の編集部が担っていたが、オンライン講座市場の急拡大によって需要が供給を大幅に上回っている。

ストアカやUdemyでオンライン講座を販売する個人事業主、企業の人材育成担当者、採用コンサルタントなど、「問題はほしいが自分で作る時間はない」というクライアント層は急増中だ。クラウドワークスやランサーズを見ると、「試験問題作成」のカテゴリには常時50〜100件以上の案件が掲載されており、単価は1問あたり500〜2,000円、まとめて100問セットの依頼では5万〜20万円という相場が確認できる。

ここにGPT-4やClaudeといったLLMの進化が重なる。2023年以前は「AIが生成した試験問題は誤りが多くて使えない」が常識だったが、Claude 3.5 Sonnet以降は専門法令や数値を含む問題でも精度が大幅に向上した。加えて、RAG（検索拡張生成）を使えば最新の法改正情報や各試験の公式シラバスをリアルタイムに参照させることができる。人間の専門家が最終確認を担う「AIファースト・人間レビュー」体制を組めば、従来の10倍以上の生産速度で問題バンクを構築できる。今がこのビジネスを立ち上げる絶好のタイミングだ。

## AIで自動化できる部分の設計

このビジネスの核心は「インプット（シラバス・教材・法令テキスト）→ 問題草案生成 → 品質チェック → 解説文生成 → 納品フォーマット整形」というパイプラインを自動化することにある。

**全体ワークフロー概要**

```
クライアントからの依頼仕様（Googleフォームで受付）
　↓
Make.com で情報を整形・トリガー発火
　↓
Claude API (claude-3-5-sonnet) で問題草案生成
　↓
GPT-4o APIで問題の論理矛盾・重複チェック（ダブルチェック）
　↓
Googleスプレッドシートに問題・選択肢・解説・難易度タグを出力
　↓
人間によるレビュー（後述）
　↓
承認済み問題をNotion DBまたはWordに整形してクライアントへ納品
```

**各AIツールの役割**

- **Claude API（claude-3-5-sonnet-20241022）**：メインの問題生成エンジン。System Promptに「あなたは〇〇資格試験の専門家です。以下のシラバス範囲から四択問題を生成してください。誤答選択肢は受験者がよく混同するパターンを採用してください」と設定し、PDFやテキストで渡した参考資料を文脈として与える。1回のAPI呼び出しで5〜10問をJSONフォーマットで出力させると後処理が楽になる。

- **GPT-4o API**：Claudeが生成した問題に対して「この問題に事実誤認・複数正解・選択肢の表現ゆれがないか批判的に検討し、問題点があれば指摘してください」というプロンプトで二次チェックを走らせる。異なるモデルに検証させることで盲点を減らす。

- **Make.com**：ワークフローのオーケストレーター。Googleフォームの送信をトリガーに、HTTP モジュールでClaude APIとGPT-4o APIを呼び出し、レスポンスをパースしてGoogleスプレッドシートに書き込む一連の処理を自動化する。

- **Notionデータベース**：問題バンクの管理に使用。タグ（試験種別・難易度・出題分野・作成日・レビューステータス）でフィルタリングでき、クライアントごとに専用のビューを作れる。

## 実際のツール設定（ステップバイステップ）

以下はMake.com＋Claude APIを使った「問題自動生成パイプライン」の構築手順だ。

**Step 1：Anthropic APIキーの取得**
1. https://console.anthropic.com にアクセスしてアカウントを作成
2. 「API Keys」セクションで新しいキーを発行し、安全な場所に保存
3. 従量課金の場合、claude-3-5-sonnetは入力1Mトークンあたり$3、出力1Mトークンあたり$15。問題10問の生成は概ね約2,000トークンなので1回約$0.05以下で収まる

**Step 2：Googleフォームで依頼仕様を受け付ける**
1. Googleフォームを新規作成し、以下の項目を設定する
   - 試験種別（例：宅建・FP2級・SPI非言語）
   - 出題範囲（例：民法・相続分野）
   - 問題形式（四択/○×/記述）
   - 必要問題数
   - 難易度指定（基礎・応用・実践）
   - 参考資料のGoogleドライブURL

**Step 3：Make.comでシナリオを組む**
1. Make.comにログインし「新しいシナリオ」を作成
2. トリガーモジュール：「Google Forms > Watch Responses」を選択し、Step 2のフォームと連携
3. HTTPモジュール①（Claude API呼び出し）を追加：
   - URL：`https://api.anthropic.com/v1/messages`
   - Method：POST
   - Headers：`x-api-key: {{APIキー}}`、`anthropic-version: 2023-06-01`、`content-type: application/json`
   - Body（JSON）：
     ```json
     {
       "model": "claude-3-5-sonnet-20241022",
       "max_tokens": 4096,
       "system": "あなたは{{試験種別}}の専門講師です。以下の条件で問題を生成し、必ずJSON配列で返してください。各要素にquestion, choices(A〜D), answer, explanation, difficultyのキーを含めること。",
       "messages": [{"role": "user", "content": "出題範囲：{{出題範囲}}、問題数：{{問題数}}問を生成してください。"}]
     }
     ```
4. JSONパースモジュールを追加してレスポンスの`content[0].text`をパース
5. Googleスプレッドシートモジュール「Add a Row」を追加：問題文・選択肢A〜D・正解・解説・難易度を各列にマッピング
6. シナリオを保存し「テスト実行」で動作確認

**Step 4：二次チェック（GPT-4o API）の組み込み**
1. Make.comのシナリオにHTTPモジュール②を追加
2. OpenAI APIエンドポイント（`https://api.openai.com/v1/chat/completions`）に先ほど生成された問題テキストを渡し「事実誤認・複数正解・曖昧な表現がないか検証し、問題点を日本語で箇条書きにしてください」とプロンプト設定
3. チェック結果をスプレッドシートの「AIレビューコメント」列に追記

**Step 5：Notionデータベースへの同期**
1. Make.comの「Notion > Create a Database Item」モジュールを追加
2. 問題・選択肢・解説・難易度・AIレビューコメント・レビューステータス（初期値「要確認」）をNotionのプロパティにマッピング
3. 以後、人間レビュアーはNotionのフィルター「レビューステータス：要確認」で未確認問題のみ表示して効率的に確認作業を行う

## 自動化できない部分（ここが重要）

AIがどれだけ優秀になっても、このビジネスで人間が担わなければならない領域がある。ここが競合との差別化ポイントであり、あなたの専門性が正当化される部分だ。

**①最新法改正・通達への対応**
宅建・社労士・FPなどの試験は毎年法改正が発生する。AIの学習データには最新改正が含まれない場合があり、2024年の民法改正や労働基準法改正を正確に反映させるには人間が最新情報をプロンプトに注入するか、生成結果を条文と照合する必要がある。

**②「ひっかけ問題」の質的評価**
誤答選択肢の巧妙さは受験者の実力を測る核心だが、AIは「表面的に似ている選択肢」は作れても「受験者が本当にひっかかるポイント」を正確に設計する能力はまだ人間に劣る。過去問分析に基づいた誤答設計は人間の判断が必要。

**③試験ごとの出題傾向・頻出テーマの戦略的配分**
「この資格試験は〇〇の分野が近年強化されている」「この企業の採用試験は非言語の難度が年々上がっている」という出題傾向の読みはドメイン専門家でないとできない。問題セットのポートフォリオ設計は人間が行う。

**④倫理・公平性の確認**
採用試験の問題は特定の属性を不利にする内容が含まれていないか（無意識のバイアスチェック）を人間が確認する必要がある。これを怠るとクライアントが法的リスクを負う。

**⑤クライアントとのコミュニケーションと要件定義**
「難易度は応用レベル」「受験者の平均正解率が60〜70%になるよう調整」といった曖昧な要望を具体的な仕様に落とし込む作業は完全に人間の仕事だ。

- 記述式問題（解説付き）：2,000〜5,000円/問
- 100問パッケージ：8万〜15万円

ランサーズでは「資格試験問題作成」カテゴリの単価は1問あたり700〜2,000円で、まとめ発注の場合は若干の値引きが一般的。

Fiverrでは「Create exam questions」カテゴリで、100問パッケージが$200〜$500（約3万〜7.5万円）で出品されているケースが多い。

## 参入のボトルネック

このビジネスには明確な参入障壁が存在する。事前に把握しておくことが失敗を避けるカギだ。

**技術面**
Make.comとClaude APIの連携は本記事の手順通りに進めれば1〜2日で構築できるが、JSONパースのエラーハンドリングやAPIのレートリミット対策など、細かいつまずきポイントが存在する。特にClaude APIのレスポンスが常にJSON形式で返ってくるとは限らないため、「構造化出力」を強制するプロンプト設計のノウハウが必要。

**専門知識面**
「AIを使えば専門知識ゼロでも問題が作れる」は誤解だ。最終レビュアーとして機能するためには、担当する資格・試験分野の基礎知識が必要。最低限、自分がその試験を合格・受験経験者レベルの知識を持つか、ドメイン専門家をレビュアーとして外注する仕組みが必要になる。

**営業・信頼構築面**
「AIで作った問題」という事実に対してクライアントが不信感を持つケースがある。初期は「AIアシスト＋人間専門家監修」という打ち出し方にして、サンプル問題セット（30問程度）を無料または低価格で提供し実績を作ることが重要。

**法的面**
クライアントが問題バンクを商業利用する際の著作権の帰属を契約書で明確にしておく必要がある。また、既存の市販問題集からの「類似問題」生成はAIが無意識に行う場合があり、著作権侵害リスクがある。プロンプトで「既存の市販問題集の問題を再現しないこと」を明示し、参照資料は公式シラバスや法令のみに限定するポリシーをクライアントに開示することが重要。

## 最初の一歩

今日から72時間以内にできる具体的な3ステップを示す。

**ステップ1：環境構築（所要時間：2〜3時間）**

まずAnthropicとOpenAIのAPIキーを取得しMake.comのアカウントを作る。
- Anthropic API登録：https://console.anthropic.com（クレジットカード登録で$5分の無料クレジット付与）
- OpenAI API登録：https://platform.openai.com
- Make.com登録：https://www.make.com（無料プランで月1,000オペレーションまで試せる）

次に、本記事のStep 3〜5を参照しながらGoogleフォーム→Make.com→Claude API→スプレッドシートの最小構成を作り、自分が詳しい資格分野（仮でよい）で10問テスト生成してみる。この「動く体験」が最初の自信になる。

**ステップ2：サンプルポートフォリオの作成（所要時間：半日）**

自分が得意な試験分野（FP・宅建・TOEIC・SPI・医療系など何でも可）で「30問＋全問解説付き」のサンプルセットをAI＋人間レビューで作成する。品質基準は「自分がこのサンプルを見て受験勉強に使いたいと思えるレベル」。このサンプルがそのまま営業ツールになる。

**ステップ3：最初の案件獲得（所要時間：継続作業）**

以下の3つのプラットフォームに同時登録し、サンプル問題を添付した提案文を送り始める。

- **クラウドワークス**（https://crowdworks.jp）：「試験問題」「資格対策」「問題作成」で検索し、予算5万円以上の案件に提案。プロフィールにサンプル問題のGoogleドライブリンクを貼る。
- **ランサーズ**（https://www.lancers.jp）：同様に「試験問題作成」カテゴリで検索・提案。サービス出品ページも作成し「30問パッケージ ¥30,000〜」のような形で掲載。
- **ストアカ**（https://www.street-academy.com）：直接クライアントになりうる「オンライン資格講座の先生」にDMで提案できる。また自分自身が「問題作成代行セミナー」を開いてノウハウを有料で教えることも副次的な収益源になる。

最初の1ヶ月は「低単価でも実績作り」を優先し、納品後に必ずクライアントに品質フィードバックをもらうこと。2〜3件こなしたレビューと実績があれば、以降は単価交渉がずっと楽になる。この業界は「信頼できる問題作成者」の絶対数が少ないため、品質さえ担保できれば継続発注率が極めて高い。まず今日、APIキー取得だけでも完了させよう。
