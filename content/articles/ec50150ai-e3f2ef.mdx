# ECサイト商品説明文・SEO最適化代行｜AI自動化で月収30万円を狙う完全解剖

---

## なぜ今なのか

日本のEC市場は2023年度に約24兆円規模に達し、経済産業省の調査では前年比約9%成長を記録している。コロナ禍をきっかけに中小企業・個人事業主がECに参入する波は止まらず、2024年現在も「楽天市場」「Amazon」「Yahoo!ショッピング」「BASE」「Shopify」への新規出店は月単位で数千件を超えるペースが続いている。

しかしここに深刻な「担い手不足」が発生している。

EC出店者の多くは商品開発・仕入れ・梱包・配送対応に追われており、**商品説明文の最適化に時間を割けない**のが実態だ。特に以下の3つの問題が顕在化している。

**①商品点数の爆発的増加**：BASEの登録ショップ数は2024年時点で200万ショップを超え、1ショップあたり平均数十〜数百点の商品を抱える。それぞれに魅力的な説明文が必要だが、1点あたりの制作コストを捻出できていない。

**②SEOアルゴリズムの複雑化**：GoogleのSGE（AI概観）導入やEEAT評価の強化により、「キーワードを詰め込むだけ」の古い手法は通用しなくなった。専門知識なしにSEO最適化された商品ページを作るのは難しい。

**③多言語・多チャネル展開のニーズ**：越境ECの普及で、日本語・英語・中国語対応の説明文が求められるケースも増えている。

ここにAIを活用した代行サービスの商機がある。GPT-4やClaudeの登場により、高品質な商品説明文の生成コストは劇的に下がった。人間がゼロから書けば1点30〜60分かかる作業が、AIと適切なプロンプト設計を組み合わせることで**5〜10分に短縮**される。この「時間差」こそがビジネスモデルの核心だ。

Upworkでは"Product Description Writing"カテゴリに年間数万件の案件が流通しており、特にShopify・Amazon向けのSEO対応説明文は需要が高い。国内のクラウドワークスでも「商品説明文作成」タスクは常時数百件が掲示されている。市場は確実に存在し、かつAI活用でスケールできる数少ない業種のひとつだ。

## AIで自動化できる部分の設計

このサービスの核となるワークフローを設計する。使用するツールは以下の通りだ。

- **Claude API（claude-3-5-sonnet）**：説明文の本文生成・リライト・トーン調整
- **GPT-4 API（gpt-4o）**：SEOキーワード抽出・競合分析補助
- **Make.com（旧Integromat）**：全体のオートメーション基盤
- **Google Sheets**：クライアントからの商品情報受け取り・納品管理
- **Ahrefs / Ubersuggest API**：SEOキーワードデータ取得
- **Notion**：プロンプトライブラリと品質チェックシートの管理

**ワークフロー全体像：**

```
クライアントが商品情報をGoogleフォームに入力
　↓
Make.comが自動検知 → Google Sheetsに記録
　↓
Make.comがGPT-4 APIにキーワード抽出タスクを投げる
　↓
抽出されたキーワードとともにClaude APIに説明文生成を依頼
　↓
生成されたテキストをGoogle Sheetsの納品シートに自動記入
　↓
品質スコアが閾値以上なら自動メール送付（未満なら人間レビューキューへ）
　↓
クライアントに納品通知メール送信
```

このフローが完成すると、**人間の介入なしに1件あたり約3〜5分で商品説明文の初稿が完成する**。

## 実際のツール設定（ステップバイステップ）

### STEP 1：Googleフォームとシートの準備

1. Googleフォームを新規作成し、以下の入力欄を設定する
   - 商品名（テキスト）
   - 商品カテゴリ（プルダウン：ファッション/家電/食品/コスメ/その他）
   - 主な特徴・スペック（段落テキスト）
   - ターゲット顧客イメージ（テキスト）
   - 出店プラットフォーム（楽天/Amazon/BASE/Shopify/その他）
   - トーン希望（プロフェッショナル/親しみやすい/高級感）
   - 競合URL（任意）

2. フォームの回答先をGoogle Sheetsにリンクし、「受注管理シート」として保存する。別シートに「納品シート」を作り、列に「商品名・生成説明文・SEOタイトル・メタディスクリプション・ステータス」を設定する。

### STEP 2：Make.comのシナリオ構築

1. Make.com（https://www.make.com）に無料登録後、新規シナリオを作成
2. **トリガー設定**：「Google Sheets」モジュールを選択 → "Watch New Rows"を設定。受注管理シートを指定し、5分ごとにポーリングするよう設定する
3. **キーワード抽出モジュール追加**：「HTTP」モジュールを追加し、OpenAI APIのエンドポイント（https://api.openai.com/v1/chat/completions）にPOSTリクエストを設定する

   リクエストボディ例：
   ```json
   {
     "model": "gpt-4o",
     "messages": [
       {
         "role": "system",
         "content": "あなたはEC向けSEOの専門家です。与えられた商品情報から、検索ボリュームが見込めるロングテールキーワードを10個抽出してください。JSON形式で返してください。"
       },
       {
         "role": "user",
         "content": "商品名：{{商品名}} カテゴリ：{{カテゴリ}} 特徴：{{特徴}}"
       }
     ]
   }
   ```

4. **Claude APIモジュール追加**：再度「HTTP」モジュールを追加し、Claude APIエンドポイント（https://api.anthropic.com/v1/messages）にPOSTリクエストを設定する。ヘッダーに`x-api-key`と`anthropic-version: 2023-06-01`を追加する

   リクエストボディ例：
   ```json
   {
     "model": "claude-3-5-sonnet-20241022",
     "max_tokens": 1500,
     "messages": [
       {
         "role": "user",
         "content": "以下の情報をもとに、{{プラットフォーム}}向けの商品説明文を作成してください。\n\n【商品名】{{商品名}}\n【特徴】{{特徴}}\n【ターゲット】{{ターゲット}}\n【SEOキーワード（必ず自然に含める）】{{GPT抽出キーワード}}\n【トーン】{{トーン}}\n\n出力形式：\n1. SEOタイトル（32文字以内）\n2. メタディスクリプション（120文字以内）\n3. 商品説明文本文（400〜600文字）\n4. 箇条書き特徴（5点）"
       }
     ]
   }
   ```

5. **Google Sheets書き込みモジュール追加**：「Google Sheets」→ "Update a Row"を設定し、Claude APIのレスポンスを納品シートの対応列に書き込む

6. **条件分岐（フィルター）設定**：生成テキストの文字数が300文字未満の場合は「要レビュー」フラグを立て、それ以外は自動承認ルートへ分岐させる

7. **Gmail通知モジュール追加**：自動承認ルートでは、クライアントのメールアドレス宛に納品完了メールを送信するよう設定。メール本文にはGoogleスプレッドシートの共有リンクを含める

### STEP 3：プロンプトのチューニング（Notionで管理）

Notionにプロンプトライブラリページを作成し、カテゴリ別（ファッション・食品・家電など）にプロンプトのバージョン管理を行う。A/Bテストの結果（クライアント満足度・修正依頼率）も記録し、月1回プロンプトを改善するサイクルを確立する。

### STEP 4：品質チェックの半自動化

生成された説明文をClaude APIに再度投入し、「SEOキーワードが自然に含まれているか」「誇大表現がないか」「プラットフォームのガイドライン違反がないか」を自動でチェックさせるレビュープロンプトを作成する。これにより人間のレビュー時間を70%削減できる。

## 自動化できない部分（ここが重要）

AIがいくら優秀でも、**人間の判断が不可欠な領域**がある。ここがあなたの差別化ポイントであり、単なる「AIの丸投げ代行」との差を生む。

**①商品の本質的な価値の見極め**
クライアントが提出した情報に「本当に売れる理由」が書かれていないケースは多い。例えば「手触りがいいタオル」という情報から「ホテル仕様の480匁・今治認定」という差別化軸を引き出すのは、ヒアリング力と商品理解があってこそだ。AIはあくまで提供された情報を処理するだけで、情報を掘り起こす作業は人間にしかできない。

**②プラットフォーム固有のルール対応**
楽天市場は商品名に「！」「☆」などの記号を使った表現が審査で引っかかることがある。Amazonは独自の禁止ワードリストを持ち、違反すると出品停止になる。こうした最新ルールの把握と、それに合わせた修正判断は人間が行う必要がある。

**③クライアントのブランドトーン調整**
高級ブランドのECサイトと、激安雑貨のフリマ系では文体が180度異なる。初回のすり合わせで「このブランドらしさ」を掴み、それをプロンプトに落とし込む作業はコミュニケーション能力が問われる。

**④A/Bテスト結果の解釈と改善提案**
楽天のRMSやShopifyのアナリティクスで「説明文変更後にCVRが上がったか下がったか」を読み取り、次の最適化方針を提案するのはデータリテラシーと仮説構築力が必要だ。AIに数字を渡せばある程度分析できるが、**クライアントのビジネス文脈を踏まえた戦略提案**は人間の仕事だ。

**⑤炎上リスク・法的表現のチェック**
「〇〇が治る」「ダイエット効果100%」などの薬機法・景品表示法違反表現はAIも生成してしまうことがある。最終チェックは必ず人間が行い、責任を持つ姿勢がクライアントの信頼につながる。

## 参入のボトルネック

正直に障壁を列挙する。

**①初期のポートフォリオ不足**
実績がない状態では価格競争に巻き込まれやすい。Upworkではプロフィール評価がなければ案件を獲得しにくく、最初の3〜5件は赤字覚悟のモニター価格で受注することを覚悟すべきだ。クラウドワークスでも同様で、最初の10〜20件はタスク単価500〜1,000円のものをこなして評価を積む必要がある。

**②APIの技術的ハードル**
Make.comの設定やAPI連携は、プログラミング未経験者には最初の2〜3日が壁になる。しかしMake.comはビジュアルで設定できるため、YouTubeの日本語チュートリアルを見ながら進めれば1週間以内にマスターできるレベルだ。完全にコーディング不要というわけではなく、JSONの読み書きや基本的なAPI仕様の理解は必要になる。

**③SEO知識の習得コスト**
「SEO最適化込み」を謳うなら、キーワード選定・内部リンク・メタデータの基礎知識は必須だ。Ahrefs（月額$99〜）やSemrush（月額$119〜）は高額なため、最初はUbersuggest（月額$12）やGoogle Search Consoleの無料機能で代替するとよい。

**④クライアントの期待値管理**
「AIが書いた文章だと思うと依頼しづらい」という心理的障壁が一部のクライアントに存在する。AIを使っていることを隠す必要はないが、「AIドラフト＋専門家レビュー」という打ち出し方をすることで信頼感を維持できる。

**⑤薬機法・景品表示法の理解**
食品・化粧品・健康器具の説明文は法律上の制約が多い。知識不足でクライアントに損害を与えた場合、損害賠償リスクがある。この分野に踏み込むなら消費者庁の公式ガイドラインを読み込むか、弁護士監修のチェックリストを活用することを強く推奨する。
